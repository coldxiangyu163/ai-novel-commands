# 写作流程指南

> 根据不同作品类型，选择合适的写作流程。不是所有命令都需要用，按需选择。
> 
> **此文件供 `/init-novel` 读取，自动生成项目定制的 `写作流程.md`**

| 题材/基调关键词 | 推荐流程 |

## 二、精简流程

**适用于**：轻松爽文、脑洞文、系统流、无脑爽、搞笑文

**命令数量**：约 10 个

### 框架设计

**必须**：
- `/design-world` → 世界观设计（简单：等级体系、基本规则）
- `/design-characters` → 人物设计（简洁模式：主角+工具人反派）

**推荐**：
- `/design-style` → 写作风格（有助于去AI味）

**跳过**：
- `/design-plot` - 不需要复杂剧情设计
- `/design-beat-sheet` - 不需要节拍表

### 大纲规划

**必须**：
- `/plan-volume` → 卷大纲

**可选**：
- `/plan-chapter` → 章细纲（也可直接写）

**跳过**：
- `/design-opening` - 轻松文不需要精细开篇设计
- `/design-climax` - 轻松文不需要精细高潮设计

### 写作执行

**必须**：
- `/auto-write` → 一键写章

**可选**：
- `/batch-write` → 批量写章（赶稿时）
- `/brainstorm-twist` → 卡文时激发创意

### 质量检查

**必须**：
- `/check-consistency` → 每10章检查一致性

**可选**：
- `/polish-chapter` → 重点章节去AI味

**跳过**：
- `/check-logic` - 轻松文不需要严密逻辑检查
- `/check-structure` - 不需要结构检查

## 九、常见问题

### Q1: 我是新手，应该用哪个流程？
**A**: 从精简流程开始，先把书写出来。等熟悉后再尝试更复杂的流程。

### Q2: 我的书是混合类型怎么办？
**A**: 以主要类型为主，按需添加其他流程的命令。

### Q3: 必须按顺序执行吗？
**A**: 开书准备必须按顺序（idea-mill → packaging-kit → init-novel）。其他阶段可以灵活调整。

### Q4: 可以跳过某些命令吗？
**A**: 可以。标注为"可选"的都可以跳过。

### Q5: learn-style 和 design-style 有什么区别？
**A**: 
- `/learn-style`：从对标作品学习文笔风格，直接生成 `写作风格.md`
- `/design-style`：根据题材自动推断风格，生成 `写作风格.md`
- **二选一即可**：如果有对标作品想模仿，用 `/learn-style`；否则用 `/design-style`
