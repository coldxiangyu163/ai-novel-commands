# 近期层动态状态更新：$ARGUMENTS

**用于初始化或同步近期层（动态状态）文件。**

> ⚠️ 此命令只处理**近期层**。全书层请使用 `/sync-memory` 命令。

---

## 分层记忆架构

```
┌─────────────────────────────────────┐
│  全书层 → /sync-memory 管理         │
│  伏笔、里程碑、套路库、感情节点...    │
├─────────────────────────────────────┤
│  卷级层 → 章节摘要自动管理           │
├─────────────────────────────────────┤
│  近期层 → 本命令管理 ⬅              │
│  人物当前状态、叙事手法记录           │
└─────────────────────────────────────┘
```

---

## 使用场景

### 场景1：初始化（init）
新项目或首次使用动态系统时，创建初始状态文件。

### 场景2：同步（sync）
已有若干章节，需要根据已写内容补全动态状态。

---

## 执行流程

### 步骤1：读取项目信息

1. `AGENTS.md` - 获取人物列表和项目结构
2. `人物档案/` - 获取所有人物的静态档案
3. 当前卷的 `章节摘要.md`（如有）
4. 最近写完的章节内容（如有）

---

### 步骤2：创建目录结构

确保项目中存在以下目录和文件：
```
动态状态/
├── 人物当前状态.md    # 各人物的动态状态
└── 叙事手法记录.md    # 最近章节的叙事手法
```

### 步骤5：同步模式（sync）专用

如果是同步已有章节，需要：

1. 读取所有已完成章节（或章节摘要）
2. 分析每章使用的叙事手法

4. 填充上述两个文件

**同步检查清单**：
- [ ] 主角当前情绪是否与最新章节一致

## 与其他命令的联动

- `/auto-write` → 写作前自动读取，写作后自动更新
- `/batch-write` → 批量写作时每章都会更新
- `/check-consistency` → 检查时验证状态一致性

---

## 手动更新场景

以下情况可能需要手动运行此命令：

1. **重大剧情转折后** - 确保状态文件准确反映变化
2. **长时间未写后恢复** - 同步记忆
3. **发现状态不一致时** - 修正错误
4. **新增重要角色后** - 添加追踪

---

## 注意事项

1. **保持简洁**：状态描述要精炼，不是重写档案
2. **及时更新**：每章写完后 auto-write 会自动更新
3. **人工可编辑**：发现AI更新不准确时可手动修正
4. **删除过期信息**：叙事记录只保留最近10章
