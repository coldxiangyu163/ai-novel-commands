# 全书大纲一致性检查

**在所有卷大纲都用 plan-volume 生成完成后，执行此命令进行全局校验。**

---

## 第一步：读取所有文件

1. `AGENTS.md` - 规划约束
2. `小说规划.md` - 总体规划
3. `全部卷大纲` - 卷一大纲.md ~ 卷X大纲.md
4. `人物档案/仇人谱.md` - 仇人清单

---

## 第二步：提取检查基准

从 AGENTS.md 中提取规划约束作为检查基准：
- 结构约束（每卷章节数）
- 等级出现时间表
- 复仇时机表
- 战力约束表
- 势力发展约束表

---

## 第三步：执行六项检查

### 3.1 伏笔闭环检查

扫描所有卷大纲，提取伏笔信息，生成全局表：

```markdown
## 伏笔追踪表

| # | 伏笔内容 | 埋设位置 | 计划回收 | 实际回收 | 状态 |
|---|----------|----------|----------|----------|------|
| 1 | XXX | 卷X第X章 | 卷X | 卷X第X章 | ✓ 已闭环 |
| 2 | XXX | 卷X第X章 | 卷X | - | ⚠ 未回收 |
| 3 | XXX | - | - | 卷X第X章 | ✗ 无埋设 |
```

**检查项**：
- [ ] 所有埋设的伏笔都有回收？
- [ ] 没有凭空出现的回收（无埋设）？
- [ ] 回收时机是否合理（不能太快/太慢）？

---

### 3.2 复仇顺序检查

对照 AGENTS.md 中的复仇时机表：

```markdown
## 复仇顺序检查

| 仇人 | 约束要求 | 实际规划 | 状态 |
|------|----------|----------|------|
| 钱富贵 | 卷二清算 | 卷X第X章 | ✓/✗ |
| 林婉儿 | 卷三清算 | 卷X第X章 | ✓/✗ |
| ... | ... | ... | ... |
```

**检查项**：
- [ ] 每个仇人的清算时机是否符合约束？
- [ ] 是否有提前清算的情况？
- [ ] 是否有遗漏未清算的仇人？

---

### 3.3 战力曲线检查

对照 AGENTS.md 中的战力约束：

```markdown
## 战力曲线检查

| 卷 | 约束上限 | 实际规划 | 增幅 | 状态 |
|----|----------|----------|------|------|
| 卷一结束 | 20m³/一阶 | Xm³/X阶 | - | ✓/✗ |
| 卷二结束 | 40m³/二阶 | Xm³/X阶 | +X% | ✓/✗ |
| ... | ... | ... | ... | ... |
```

**检查项**：
- [ ] 每卷结束时战力是否符合约束上限？
- [ ] 是否有单卷升级超过一阶的情况？
- [ ] 战力曲线是否平滑（无突然暴涨）？

---

### 3.4 势力发展检查

对照 AGENTS.md 中的势力约束：

```markdown
## 势力发展检查

| 卷 | 约束规模 | 实际规划 | 状态 |
|----|----------|----------|------|
| 卷一 | 5人核心 | X人 | ✓/✗ |
| 卷二 | 5-15人 | X人 | ✓/✗ |
| ... | ... | ... | ... |
```

**检查项**：
- [ ] 每卷团队规模是否符合约束？
- [ ] 势力发展是否过快？

---

### 3.5 等级出现检查

对照 AGENTS.md 中的等级时间表：

```markdown
## 等级出现检查

| 等级 | 约束时间 | 首次出现 | 状态 |
|------|----------|----------|------|
| F级 | D-Day | 卷X第X章 | ✓/✗ |
| E级 | D+30 | 卷X第X章 | ✓/✗ |
| D级 | D+90 | 卷X第X章 | ✓/✗ |
| ... | ... | ... | ... |
```

**检查项**：
- [ ] 各等级首次出现是否符合时间表？
- [ ] 是否有等级出现过早的情况？

---

### 3.6 卷间衔接检查

检查相邻卷之间的衔接：

```markdown
## 卷间衔接检查

| 衔接点 | 前卷结束状态 | 后卷开始状态 | 状态 |
|--------|--------------|--------------|------|
| 卷一→卷二 | 主角X状态，团队X人 | 主角X状态，团队X人 | ✓ 顺畅 / ⚠ 断层 |
| 卷二→卷三 | ... | ... | ... |
| ... | ... | ... | ... |
```

**检查项**：
- [ ] 每卷开始状态是否与前卷结束一致？
- [ ] 是否有剧情跳跃或断层？
- [ ] 时间线是否连贯？

---

### 3.7 感情线发展检查（如有感情线）

对照感情线规划表：

```markdown
### 女主出场检查
| 卷 | 规划出场章节 | 实际出场章节 | 戏份比重 | 状态 |
|----|--------------|--------------|----------|------|
| 卷一 | 第X-X章 | 第X-X章 | X% | ✓/✗ |
| 卷二 | 第X-X章 | 第X-X章 | X% | ✓/✗ |
| ... | ... | ... | ... | ... |

### 感情名场面检查
| 场面 | 规划位置 | 实际位置 | 状态 |
|------|----------|----------|------|
| 初遇 | 卷X第X章 | 卷X第X章 | ✓/✗ |
| 心动时刻 | 卷X第X章 | 卷X第X章 | ✓/✗ |
| 表白/确认 | 卷X第X章 | 卷X第X章 | ✓/✗ |

### 感情冲突闭环检查
| 冲突 | 发生卷 | 规划解决卷 | 实际解决卷 | 状态 |
|------|--------|-----------|-----------|------|
| [冲突1] | 卷X | 卷X | 卷X | ✓ 已解决 / ⚠ 未解决 |

```

**检查项**：
- [ ] 感情发展阶段是否符合规划？
- [ ] 是否有感情发展过快/过慢的情况？
- [ ] 女主戏份是否均衡分布？
- [ ] 感情名场面是否都有安排？
- [ ] 感情冲突是否有合理解决？
- [ ] 感情线与主线是否有机融合？

---

### 3.8 详略节奏检查

检查全书的详略分布是否合理：

```markdown
## 详略节奏检查

### 高潮场景分布
| 卷 | 高潮场景数 | 位置 | 间隔 | 状态 |
|----|-----------|------|------|------|
| 卷一 | X个 | 第X、X、X章 | X章/个 | ✓ 合理 / ⚠ 过密/过疏 |
| 卷二 | X个 | 第X、X、X章 | X章/个 | ✓/⚠ |
| ... | ... | ... | ... | ... |

### 节奏波动检查
| 卷 | 节奏问题 |
|----|----------|
| 卷一 | ✓ 节奏合理 / ⚠ 开头过慢 / ⚠ 中段拖沓 / ⚠ 高潮连续 |
| 卷二 | ... |
| ... | ... |

### 详略分布建议
- 需要扩写的场景：[列出]
- 需要精简的场景：[列出]
- 需要补充过渡的位置：[列出]
```

**检查项**：
- [ ] 每卷是否有足够的高潮场景？（建议每卷2-3个大高潮）
- [ ] 高潮场景间隔是否合理？（建议间隔3-5章）
- [ ] 是否存在连续多章都是高潮或都是平淡？
- [ ] 每卷开头是否有钩子？
- [ ] 每卷结尾是否有悬念？
- [ ] 日常/过渡场景是否过多？（建议不超过30%）

## 第五步：问题清单与修改建议

如果有问题，列出详细清单：

```markdown
## 需要修改的问题

### 问题1：[伏笔未回收]
- 位置：卷X第X章埋设的"XXX"
- 问题：计划卷X回收，但实际未找到回收点
- 建议：在卷X第X章增加回收场景

### 问题2：[复仇顺序违反]
- 位置：卷X第X章
- 问题：赵磊应在卷四清算，实际在卷三第X章清算
- 建议：将赵磊清算移至卷四，卷三改为铺垫

### 问题3：[战力超标]
- 位置：卷X结束
- 问题：空间达到Xm³，超过约束上限Xm³
- 建议：降低本卷升级幅度，或调整约束

...
```

---

## 第六步：保存报告

1. 保存检查报告到 `质量检查/全书一致性报告.md`
2. 保存伏笔追踪表到 `质量检查/伏笔追踪表.md`
3. 如有问题，提示用户修改对应卷大纲

---

## 使用时机

- ✓ 所有卷大纲（plan-volume）完成后
- ✓ 修改某卷大纲后，重新检查
- ✓ 写作过程中发现问题，检查是否影响全局
- ✗ 不要在只完成部分卷时使用（信息不完整）
