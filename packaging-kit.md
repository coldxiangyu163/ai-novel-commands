# 包装生产线（点击率优先）：$ARGUMENTS

你要做的是“让读者点进来并继续读”，不是写论文。

本命令要求你输出**可直接投放/发布**的包装物：
- 书名候选（20个）
- 简介（6版）
- 标签/关键词（用于搜索与推荐）
- 封面方向文案（给美工/封面生成用）
- 开篇钩子方案（前三章节奏+第一章前500字）

默认平台：**番茄**（如用户说明起点则按起点偏好调整）。

---

## 0）输入检查（信息不全也要先产出）

### 0.1 检测灵感方案文件（自动衔接 idea-mill）

先检测当前目录是否存在 `灵感方案.md`（由 `/idea-mill` 生成）：

**如果存在**，自动读取并提取：
| 提取项 | 来源 |
|--------|------|
| 一句话卖点 | 灵感方案 → 选中方案 → 一句话卖点 |
| 题材 | 灵感方案 → 类型信号 → 题材 |
| 爽点类型 | 灵感方案 → 类型信号 → 爽点类型 |
| 核心机制 | 灵感方案 → 核心设定 → 核心机制 |
| 核心对手 | 灵感方案 → 核心设定 → 核心对手/压力源 |
| 主角反差 | 灵感方案 → 核心设定 → 主角反差 |
| 三章方案 | 灵感方案 → 三章兑现方案（作为开篇方案的参考基准）|
| 扩写骨架 | 灵感方案 → 50万字扩写骨架 |

**输出确认**：
```markdown
## 检测到灵感方案，已自动填充以下信息：

- 一句话卖点：XXX
- 题材：XXX
- 爽点类型：XXX
- 核心机制：XXX
- 主角反差：XXX

### 三章兑现方案（来自 idea-mill，将作为开篇方案的参考基准）
- 第1章：XXX
- 第2章：XXX
- 第3章：XXX

将基于以上信息生成包装方案，开篇钩子会**保持与三章方案一致**。
```

**如果不存在灵感方案文件**，继续解析 `$ARGUMENTS`。

### 0.2 解析用户输入

先解析 `$ARGUMENTS`：
- 如果包含一句话卖点/设定/书名草案：直接进入产出。
- 如果 `$ARGUMENTS` 为空或过于模糊：只问**最小必要信息**（最多6个问题），其余用合理假设继续产出。

最小必要信息（优先问这6个）：
1. 平台：番茄/起点（默认番茄）
2. 题材赛道（男频爽文/女频爽文/悬疑等）
3. 主角身份反差（“表面X，实际/将成为Y”）
4. 金手指/核心机制（一句话讲清）
5. 核心目标（主角想要什么）
6. 核心冲突/代价（达成目标要付出什么/对手是谁）

额外加分信息（若用户提供就用）：受众画像、禁区/审核敏感点、是否多女主、主线爽点类型、对标作品。

## 2）书名工厂（20个 + 评分筛选）

### 2.1 书名生成规则
- 必须有**题材信号**（读者一眼知道是什么爽）。
- 必须有**可搜索性**（关键词明确，别写“空泛诗意”。）
- 必须有**点击诱因**（反差/悬念/承诺/身份）。
- 避免生僻字、难记、容易撞名。

### 2.2 输出：20个候选书名
按不同风格分组输出，每个书名附：关键词、卖点角度、风险提示。

```markdown
## 书名候选（20）

### A. 强类型信号（更番茄）
1. 《》——关键词：；角度：；风险：
...

### B. 身份反差/扮猪吃虎
...

### C. 悬念钩子/设定党
...

### D. 轻松反差/梗感
...
```

### 2.3 书名评分表（选Top5）

```markdown
## 书名评分（Top5推荐）
评分维度：
- 类型信号（0-2）
- 差异化（0-2）
- 搜索友好（0-2）
- 点击欲（0-2）
- 审核风险（0到-2）

| 排名 | 书名 | 信号 | 差异 | 搜索 | 点击 | 风险 | 总分 | 一句话解释 |
|---|---|---:|---:|---:|---:|---:|---:|---|
| 1 | 《…》 |  |  |  |  |  |  |  |
...
```

## 3）简介工厂（6版，直接可用）

番茄简介要求：短句、强承诺、强冲突、少背景解释。

输出 6 种不同风格（同一卖点不同表达），每版控制在可直接发布的长度：
- V1：爽点直给（最番茄）
- V2：设定机制（系统/规则/能力）
- V3：悬疑钩子（谜团/真相/身份）
- V4：情感驱动（若有感情线）
- V5：高压危机（末世/生存/追杀/时间限制）
- V6：轻松梗感（喜剧爽文可用）

```markdown
## 简介（6版）

### V1（爽点直给）
[直接可用文本]

### V2（设定机制）
...
```

并给“首句钩子”备选 10 条（简介第一句/第一行用）：

```markdown
## 简介首句钩子（10条）
1. 
2. 
...
```

## 6）开篇钩子包（前三章节奏 + 第一章前500字）

**重要**：如果检测到 `灵感方案.md`，开篇方案必须**以其三章兑现方案为基准**，在此基础上优化包装表达，而非另起炉灶。

输出 3 套不同开篇方案（为了 A/B 测试），每套必须包含：
- 前 100 字：第一句 + 冲突/反差 + 悬念
- 前 500 字：信息释放顺序（不要讲背景）
- 第一章章末钩子（危机/反转/期待三选一）
- 第二章任务（建立目标+展示能力）
- 第三章任务（第一个小高潮/兑现承诺）

**如果有灵感方案**：
- 方案1：严格遵循灵感方案的三章设计，仅优化表达
- 方案2：在灵感方案基础上微调节奏
- 方案3：保留核心但尝试不同切入角度

```markdown
## 开篇方案（3套）

### 方案1（基于灵感方案）：
- 前100字钩子：
- 前500字信息序：
- 第1章末钩子：
- 第2章任务：
- 第3章小高潮：

### 方案2（节奏微调）：
...

### 方案3（切入角度变化）：
...
```
