# 人物深度设计

**在 init-novel 生成基础框架后，根据作品需求设计人物体系。**

> **核心原则**：按需设计，不同题材/基调需要不同深度的人物设计。

---

## 第一步：读取项目文件并评估需求

读取以下文件：
1. `AGENTS.md` - 获取题材、基调、主角基础信息
2. `小说规划.md` - 获取感情线比重、配角预设

### 1.1 根据作品类型确定设计深度

| 作品类型 | 主角设计 | 反派设计 | 感情线 | 配角设计 | 关系网 |
|----------|----------|----------|--------|----------|--------|
| **轻松爽文/脑洞文** | 简洁（性格+红线） | 工具人级别 | 点缀/无 | 功能性 | 不需要 |
| **复仇流/虐主流** | 深度（含创伤） | 完整仇人谱 | 按需 | 按需 | 简单 |
| **权谋/宫斗** | 深度 | 多层次反派 | 复杂 | 群像级 | 必须 |
| **言情/甜宠** | 中等 | 简单 | 核心 | 配角CP | 感情网 |
| **群像剧** | 多主角 | 复杂 | 多线 | 深度 | 必须 |

**询问用户**：
```
根据您的作品（[题材] + [基调]），建议采用 [X] 级别的人物设计。

需要设计：
- 主角：[简洁/中等/深度]
- 反派：[工具人/仇人谱/多层次]
- 感情线：[无/点缀/重要/核心]
- 配角：[功能性/有特色/群像级]

是否同意？或者您有特殊需求？
```

---

## 第二步：主角设计

### 2.1 必须设计（所有作品）

```markdown
## 主角档案

### 基本信息
- 姓名：
- 金手指：
- 核心目标：

### 性格关键词（3个）
1. [关键词1]：表现...
2. [关键词2]：表现...
3. [关键词3]：表现...

### 人设红线
- ❌ 禁止：[行为1]
- ❌ 禁止：[行为2]
- ✓ 必须：[行为1]
- ✓ 必须：[行为2]
```

### 2.2 可选设计（根据需求）

**深度背景**（复仇流/虐主流需要）：
- 背景故事（500-1000字）
- 性格成因
- 内心创伤/执念

**人物弧光**（有成长线的作品需要）：
- 起点状态 → 终点状态
- 关键转变节点

**说话风格**（需要鲜明人设的作品）：
- 口头禅
- 说话特点
- 内心独白风格

---

## 第三步：反派/对手设计

### 3.1 根据作品类型选择模式

**模式A：工具人反派**（轻松爽文）
```markdown
## 反派工具人列表

| 反派 | 功能 | 出场卷 | 结局 |
|------|------|--------|------|
| [名字] | 送装备/送经验 | 卷X | 被打脸 |
| [名字] | 制造冲突 | 卷X | 被碾压 |
```

**模式B：仇人谱**（复仇流）
```markdown
## 仇人谱

### 仇恨来源
[简述仇恨起源]

### 仇人清单
| 仇人 | 仇恨等级 | 清算卷 | 清算方式 |
|------|----------|--------|----------|
| [名字] | ⭐⭐⭐ | 卷X | [方式] |
| [名字] | ⭐⭐⭐⭐⭐ | 卷X | [方式] |
```

**模式C：多层次反派**（权谋/复杂剧情）
- 需要为每个重要反派设计完整档案
- 包含动机、手段、与主角的关系变化

---

## 第四步：感情线设计（如需要）

### 4.1 根据比重选择模式

**点缀级（10-20%）**：
```markdown
## 感情线（点缀）

- 女主：[名字]
- 人设：[一句话描述]
- 功能：[在剧情中的作用]
- 互动风格：[甜/搞笑/损友...]
```

**重要级（30-50%）**：
- 需要女主详细档案
- 需要感情发展时间线
- 需要关键感情节点

**核心级（50%+）**：
- 需要完整的感情线设计
- 需要感情冲突设计
- 需要双视角发展

---

## 第五步：配角设计（如需要）

### 5.1 根据作品类型选择模式

**功能性配角**（大多数爽文）：
```markdown
## 配角列表

| 配角 | 功能 | 与主角关系 | 特点 |
|------|------|------------|------|
| [名字] | 捧哏/震惊 | 好友 | [一个特点] |
| [名字] | 提供信息 | 盟友 | [一个特点] |
```

**有特色配角**（需要配角出彩）：
- 为重要配角设计1-2个闪光点
- 设计配角的独特价值

**群像级配角**（群像剧/权谋）：
- 每个核心配角需要完整档案
- 需要配角间的关系设计

---

## 第六步：人物关系网（复杂作品需要）

仅在以下情况需要设计关系网：
- 权谋/宫斗类
- 群像剧
- 多势力对抗
- 复杂感情线

```markdown
## 人物关系简图

主角
├── 盟友：[名字]、[名字]
├── 对手：[名字]、[名字]
├── 感情：[名字]
└── 仇人：[名字]、[名字]
```

---

## 第七步：输出文件

根据实际设计内容生成文件：

```
人物档案/
├── 主角.md          # 必须
├── [反派].md        # 如有重要反派
├── [女主].md        # 如有感情线
├── 配角列表.md      # 如有配角设计
└── 关系图.md        # 如有关系网设计
```

---

## 第八步：输出确认

```
===== 人物设计完成 =====

设计深度：[简洁/中等/深度]

已生成文件：
✓ 人物档案/主角.md
✓ [其他根据需求生成的文件]

下一步：
/design-style  → 设计写作风格
或
/design-plot   → 深度设计剧情
===========================
```

---

## 注意事项

1. **按需设计**：不要为了完整而设计不需要的内容
2. **服务剧情**：人物设计是为剧情服务的
3. **轻松作品轻松设计**：脑洞爽文不需要复杂的人物关系网
4. **保持一致**：人设红线在全书中必须遵守
5. **留有余地**：不要把所有细节都定死
