# 深度剧情设计

**在 init-novel 生成框架后，深入设计剧情结构，为 plan-volume 提供剧情蓝图。**

> **核心理念**：
> - 大纲决定上限，写作只是执行
> - 先想清楚"讲什么故事"，再考虑"怎么讲"
> - 一个平庸的骨架，再精彩的场景设计也救不回来

---

## 第一步：读取项目文件

1. `AGENTS.md` - 写作规范、人设、约束
2. `小说规划.md` - 总体规划、卷章结构
3. `世界观/` - 世界观设定（等级体系、势力分布、时间线等）
4. `人物档案/` - 主角、仇人、配角档案
5. `人物档案/[角色名]-内核档案.md` - 角色内核（Want/Need/Lie/Wound/Truth，如已创建）

> 注：按推荐流程，design-plot 在 design-world 和 design-characters 之后执行，
> 此时世界观和人物档案应该已经完善。

## 第三步：核心概念提炼（High Concept）

> **如果你不能用一句话说清楚这个故事为什么吸引人，那它可能真的不够吸引人。**

### 3.1 全书一句话概念

**模板**：「一个[什么样的人]，在[什么情境下]，必须[做什么]，否则[会怎样]」

**好的例子**：
- 《斩神》：一个普通高中生，发现诸神真实存在且在奴役人类，他选择弑神
- 《诡秘之主》：一个穿越者，在蒸汽朋克+克苏鲁世界，必须扮演不同身份升级，否则会被污染

```markdown
## 全书核心概念

【一句话】：
_________________________________

【吸引力三要素】：
- 新奇感：[有什么是读者没见过的？]
- 代入感：[读者为什么会在乎主角？]
- 期待感：[读者最想看到什么？]
```

### 3.2 概念独特性检验

| 检验项 | 是/否 | 说明 |
|--------|-------|------|
| 与同类型作品有差异化？ | | |
| 能在3秒内让读者产生好奇？ | | |
| 有"如果是我会怎么做"的代入空间？ | | |
| 有可以持续挖掘的深度？ | | |

## 第四步：设计戏剧结构

### 4.1 三幕结构设计

```markdown
## 三幕结构

### 第一幕：建置（约占25%，卷一~卷二/三）
- **开场状态**：主角的初始状态
- **触发事件**：打破平衡的事件
- **第一转折点**：主角被迫踏上旅程
- **目标确立**：主角明确要做什么

### 第二幕：对抗（约占50%，卷三~卷七）
- **上升动作**：主角逐步积累力量
- **中点转折**：重大发现/逆转，游戏规则改变
- **危机加深**：反派反击，主角陷入困境
- **第二转折点**：最黑暗时刻，似乎一切都完了

### 第三幕：解决（约占25%，卷八~卷十）
- **最终对决**：主角与最终反派的决战
- **高潮**：最大的冲突和情感释放
- **结局**：新的平衡建立
- **尾声**：收束伏笔，展望未来
```

### 4.2 五幕结构设计（可选，更细致）

```markdown
## 五幕结构

### 第一幕：阐述（卷一~卷二）
- 世界观展示
- 人物介绍
- 冲突种子埋下

### 第二幕：上升（卷三~卷四）
- 冲突逐渐展开
- 主角开始行动
- 小胜利积累

### 第三幕：高潮（卷五~卷六）
- 中期大高潮
- 重大转折
- 局势逆转

### 第四幕：下降（卷七~卷八）
- 反派反击
- 主角遭遇重大挫折
- 黑暗时刻

### 第五幕：结局（卷九~卷十）
- 最终决战
- 大仇得报
- 新秩序建立
```

## 第五步：设计节拍表（Blake Snyder's Beat Sheet）

> **核心理念**：
> - 戏剧结构是"骨架"，节拍表是"关节定位"
> - 节拍表将抽象的三幕结构转化为具体的章节位置
> - 每个节拍都有明确的功能，对应特定的剧情功能和情感效果

### 5.1 调用节拍表系统

**运行命令**：
```
/design-beat-sheet
```

这个命令会：
1. 基于全书总章节数（从 `小说规划.md` 读取）计算节拍位置
2. 生成包含15个关键节拍的节拍表
3. 输出 `节拍表.md` 和 `节拍表速查.md`

### 5.2 节拍表与戏剧结构对应关系

| 节拍 | 百分比位置 | 对应戏剧结构 | 功能 |
|------|----------|------------|------|
| 开场意象 | 0-1% | 第一幕：建置 | 定调，展示主角初始状态 |
| 主题陈述 | 5% | 第一幕：建置 | 点出全书主题 |
| 铺垫 | 1-10% | 第一幕：建置 | 建立日常世界 |
| 触发事件 | 10% | 第一转折点 | 打破平衡，主线启动 |
| 争论 | 10-20% | 第一幕后段 | 主角犹豫，是否踏上旅程 |
| 第二幕开始 | 20% | 第二幕：对抗 | 主角真正踏上旅程 |
| B故事 | 22% | 第二幕前段 | 感情线/成长线开启 |
| 游戏时刻 | 20-50% | 第二幕前半 | 展示类型/题材的核心魅力 |
| 中点 | 50% | 第二幕中心 | 重大反转，游戏规则改变 |
| 坏人逼近 | 50-75% | 第二幕后半 | 反派反击，压力升级 |
| 失去一切 | 75% | 第二转折点 | 最黑暗时刻，似乎一切都完了 |
| 灵魂的黑夜 | 75-80% | 第三幕前段 | 内心挣扎，找到突破方法 |
| 第三幕开始 | 80% | 第三幕：解决 | 主角重整旗鼓，反击开始 |
| 决战 | 80-99% | 第三幕高潮 | 最终对决 |
| 结局意象 | 99-100% | 第三幕尾声 | 新的平衡，呼应开场 |

### 5.3 节拍表设计要点

**必须有足够张力的关键节拍**：
- 触发事件（10%）：≥★★★☆☆张力
- 中点（50%）：≥★★★★☆张力，必须是重大反转
- 失去一切（75%）：≥★★★★☆张力，最黑暗时刻
- 决战（90%）：≥★★★★★张力，全书最高潮

**节拍表与后续工具的联动**：
- 节拍位置 → `plan-volume` 时必须对应高潮章节
- 节拍位置 → `design-explosion-curve` 爆点必须对接
- 节拍位置 → `design-rhythm-master` 检验张力是否足够

## 第六步：设计主线冲突链（三维冲突模型）

> **核心理念**：
> - 好故事的本质是冲突，冲突不仅是"打架"
> - 三维冲突：外部冲突（人vs人/环境）+ 人际冲突（人vs人际关系）+ 内心冲突（人vs自己）
> - 三维冲突叠加的章节，爆发力是单维冲突的3倍

### 6.1 核心冲突定义（三维模型）

```markdown
## 核心冲突（三维模型）

### 第一维：外部冲突（人vs人/环境）
**定义**：主角与外部世界的对抗
- 主要对手：XXX（如：仇人/反派势力/丧尸/末世环境）
- 冲突本质：XXX（如：生存资源争夺/权力斗争/复仇）
- 外部冲突类型：
  - 敌人追杀/对抗
  - 资源争夺
  - 生存危机
  - 环境威胁
  - 势力战争

### 第二维：人际冲突（人vs人际关系）
**定义**：主角与队友/盟友/亲人的关系冲突
- 主要冲突关系：主角 vs XXX（如：队友/女主/家人）
- 冲突类型：
  - 理念分歧（如：主角冷酷vs队友理想主义）
  - 信任危机（如：队友被怀疑是内鬼）
  - 利益冲突（如：资源分配矛盾）
  - 背叛/误会
  - 权力斗争
- 人际冲突作用：增加剧情层次，深化人物关系

### 第三维：内心冲突（人vs自己）
**定义**：主角内心的挣扎和抉择
- 主角内心挣扎：XXX（如：冷酷vs人性残余/复仇vs放下）
- 需要克服的弱点：XXX（如：恐惧/自私/仇恨）
- 两难抉择：XXX vs XXX（如：救一人还是救百人？）
- 内心冲突类型：
  - 价值观冲突（如：生存vs道德）
  - 两难抉择（如：主线目标vs保护所爱）
  - 恐惧克服（如：面对创伤）
  - 信念动摇（如：对目标的怀疑）
  - 黑化边缘（如：为力量付出代价）

### 三维冲突叠加示例
**单维冲突场景**（张力★★☆☆☆）：
> 主角被仇人追杀，逃跑。

**三维冲突场景**（张力★★★★★）：
> 主角被仇人追杀（外部冲突），队友因害怕要撤退与主角争吵（人际冲突），主角内心挣扎：是牺牲队友独自逃生，还是为了队友冒险反击？（内心冲突）

### 主题冲突（贯穿全书的理念对立）
- 对立理念：XXX vs XXX（如：极致利己 vs 虚伪圣母）
- 理念如何体现在冲突中：[简述]
```

### 6.2 冲突递进链（递进设计）

```markdown
## 冲突递进链

| 阶段 | 卷 | 冲突对象 | 冲突等级 | 赌注 |
|------|----|---------|---------|----|
| 个人生存 | 卷一~二 | 小人/杂鱼 | ★☆☆☆☆ | 物资/安全 |
| 小团队冲突 | 卷三~四 | 中等势力 | ★★☆☆☆ | 据点/队友 |
| 势力对抗 | 卷五~六 | 大势力 | ★★★☆☆ | 势力存亡 |
| 安全区斗争 | 卷七~八 | 高层权贵 | ★★★★☆ | 秩序/真相 |
| 最终对决 | 卷九~十 | 终极反派 | ★★★★★ | 人类命运 |
```

## 第五步（进阶）：设计双轨剧情架构（A/B故事）

> **A故事（外部线/情节线）**：
> - **定义**：主角要达成的具体物理目标（如：复仇、夺冠、求生）。
> - **核心驱动力**：欲望与恐惧。
> - **关注点**：爽感、节奏、悬念。

> **B故事（内部线/关系线）**：
> - **定义**：主角的内心成长或关键关系变化（如：学会信任、与父亲和解）。
> - **核心驱动力**：情感与信念（承接 design-theme）。
> - **关注点**：共鸣、泪点、人物弧光。

### 双线交叉节点（The Cross-Over）
| 节点 | 位置 | A故事状态 | B故事状态 | 相互影响 |
|------|------|----------|----------|---------|
| **激励事件** | 卷一 | A故事开启 | B故事萌芽 | 为了[A目标]，不得不面对[B关系] |
| **中点转折** | 卷X | A故事重大转折 | B故事深化 | A的胜利导致B的危机（或反之） |
| **一无所有** | 卷X | A故事彻底失败 | B故事濒临破裂 | 只有解决了B（内心），才能解决A（外部） |
| **高潮决战** | 卷X | A故事终极对决 | B故事完成升华 | 用B的领悟，实现A的胜利 |

### 全书核心抉择（终极二选一）

> **封神法则**：最精彩的故事，高潮时主角必须在A和B之间做出撕裂性的选择。
> 如果主角能轻松两全，故事就缺乏深度；如果必须牺牲，才能触及灵魂。

```markdown
## 全书核心抉择设计

### 终极二选一设定
| 项目 | 内容 |
|------|------|
| **A线终极目标** | [如：杀死终极反派 / 登顶最强 / 拯救世界] |
| **B线终极目标** | [如：保护女主 / 守住人性 / 维系羁绊] |
| **互斥点** | [为什么不能两全？如：反派绑架女主 / 终极力量需要牺牲同伴] |
| **抉择时刻** | 卷X第X章 |

### 主角的选择路径

**选项分析**：
| 选项 | 选择内容 | 获得 | 失去 | 情感效果 |
|------|----------|------|------|----------|
| 选A弃B | 优先外部目标 | [A的胜利] | [B的代价] | 震撼/遗憾 |
| 选B弃A | 优先内心/关系 | [B的守护] | [A的未竟] | 感动/意难平 |
| 第三条路 | 找到两全方案 | [A+B] | [更大的代价] | 燃+虐 |

**本书选择**：
- 主角最终选择：____
- 付出的代价：____
- 这个选择体现的主题：____

### 分卷抉择递进（小抉择铺垫大抉择）

> 终极抉择不能凭空出现，需要通过分卷的"小抉择"逐步建立模式

| 卷 | A线目标 | B线目标 | 小互斥点 | 主角选择 | 代价 |
|----|---------|---------|----------|----------|------|
| 卷二 | [小目标] | [小关系] | [小冲突] | 选A | [小代价] |
| 卷四 | [中目标] | [中关系] | [中冲突] | 两难后选B | [中代价] |
| 卷六 | [大目标] | [大关系] | [大冲突] | 艰难选择 | [大代价] |
| 卷八 | [终极目标] | [终极关系] | [终极冲突] | **终极抉择** | [终极代价] |

### 抉择设计检验

| 检验项 | 状态 |
|--------|------|
| 互斥点合理（不是人为制造的傻逼困境）？ | |
| 两个选项都有道理（不是明显的对错题）？ | |
| 代价是真实的（不是假代价、事后反转）？ | |
| 选择体现主角的核心价值观？ | |
| 选择能触动读者（而不是"早该这样"）？ | |
| 有分卷小抉择作为铺垫？ | |
```

### 抉择类型参考

| 类型 | 说明 | 经典案例 |
|------|------|----------|
| **目标vs关系** | 完成使命还是保护所爱 | 为救世界牺牲爱人 |
| **复仇vs放下** | 继续仇恨还是选择原谅 | 发现仇人也是受害者 |
| **力量vs人性** | 获得力量的代价是失去自我 | 黑化换取实力 |
| **个人vs集体** | 自己的幸福还是更多人的利益 | 牺牲小我成全大我 |
| **真相vs安宁** | 揭露真相还是维持现状 | 真相会毁掉一切 |
| **自由vs责任** | 追求自由还是承担责任 | 逃离还是守护 |

## 第七步：设计爆点递进曲线

> **核心理念**：
> - 爆点是让读者"wow"的瞬间，是追读的核心动力
> - 爆点需要系统规划，不能靠临时发挥
> - 爆点需要递进：后期爆点必须比前期更强

### 7.1 调用爆点曲线系统

**运行命令**：
```
/design-explosion-curve
```

这个命令会：
1. 系统化规划全书爆点分布（建议100-150个爆点）
2. 将爆点分为6大类型：打脸、反转、揭秘、觉醒、虐心、甜蜜
3. 设计爆点强度递进（★~★★★★★五星评级）
4. 检测爆点间隔（不超过15章无中等以上爆点）
5. 生成 `爆点曲线.md` 和 `爆点速查表.md`

### 7.2 爆点六大类型

| 类型 | 定义 | 典型场景 | 情绪效果 |
|------|------|---------|----------|
| **打脸** | 他人轻视主角，主角展示实力反转 | 扮猪吃虎/装逼打脸/身份揭露 | 爽 |
| **反转** | 剧情/信息的意外逆转 | 盟友是敌人/真相揭示/计中计 | 震撼 |
| **揭秘** | 重大悬念/世界观真相揭晓 | 身世揭晓/末世起源/主角前世真相 | 好奇心满足 |
| **觉醒** | 主角突破瓶颈/觉醒新能力 | 升级突破/系统解锁/血脉觉醒 | 燃 |
| **虐心** | 主角/读者喜爱的角色遭遇不幸 | 队友牺牲/女主被抓/背叛 | 虐（需后续释放） |
| **甜蜜** | 感情线的高光时刻 | 表白/确认关系/并肩作战/英雄救美 | 甜/心动 |

### 7.3 爆点与节拍表对接

**关键节拍必须有爆点**：

| 节拍 | 位置 | 必备爆点类型 | 强度 |
|------|------|-------------|------|
| 触发事件（10%） | 第X章 | 觉醒/揭秘 | ★★★☆☆ |
| 第二幕开始（25%） | 第X章 | 打脸/觉醒 | ★★★☆☆ |
| 中点（50%） | 第X章 | 反转/揭秘 | ★★★★☆ |
| 失去一切（75%） | 第X章 | 虐心 | ★★★★☆ |

### 7.4 爆点递进原则

**递进公式**：
```
前期（卷1-3）：以★★☆☆☆~★★★☆☆为主
中期（卷4-7）：以★★★☆☆~★★★★☆为主
后期（卷8-10）：以★★★★☆~★★★★★为主
```

**同类型爆点必须递进**：
- 第一次打脸：打小人物（★★☆☆☆）
- 第二次打脸：打中层势力（★★★☆☆）
- 第三次打脸：打大势力（★★★★☆）
- 最终打脸：打终极反派（★★★★★）

## 第十三步：剧情张力增强方案（反平淡策略）

> **重要**：为了避免剧情流水账和反派脸谱化，请设计以下"调料"：

```markdown
## 剧情张力增强方案

### 1. 蝴蝶效应（打破全知全能）
- **核心理念**：主角重生改变了历史，导致前世经验失效或产生负面后果。
- **设计**：
  - [事件]：主角做了什么改变
  - [后果]：导致了什么意料之外的危机
  - [应对]：主角如何绝地求生

### 2. 亦敌亦友的竞争者（提升反派魅力）
- **角色设定**：[姓名/性格]（非黑即白，有原则的对手）
- **互动模式**：竞争资源 -> 暂时联手 -> 互相惜惺
- **作用**：增加人物关系复杂度

### 3. 真实的危机（拒绝假危机）
- **核心理念**：重要角色（如女主/兄弟）不仅是被保护对象，也会受伤、牺牲或面临残酷抉择。
- **设计**：
  - [危机事件]：如被控制、残疾、被迫背叛
  - [抉择]：角色做出的牺牲/主角付出的代价

### 4. 两难抉择设计（灵魂拷问场景）★★★ 重点
- **核心理念**：两难抉择是深化人物的最佳工具，选择比能力更能展现人格。
- **使用命令**：
  ```
  /design-dilemma "场景描述"
  ```
- **两难抉择的核心**：
  - 无论选哪个都要付出重大代价
  - 没有"两全其美"的第三选项
  - 选择体现主角价值观和成长
- **典型两难模板**：
  - 救人vs完成任务（人性 vs 理性）
  - 真相vs稳定（正义 vs 稳定）
  - 女主vs队友（爱情 vs 责任）
  - 底线vs生存（道德 vs 生存）
  - 复仇vs放下（复仇 vs 放下）
- **关键节点使用**：
  - "灵魂的黑夜"节拍（75-80%）：必须有重大两难抉择
  - 各卷高潮：建议有中等两难抉择
  - 分卷小抉择铺垫终极抉择
- **设计要点**：
  - 代价要真实，不能轻易挽回
  - 挣扎要充分，跨2-3章展示
  - 选择要自主，不能被他人替代
  - 与全书主题对接

### 5. 人性灰度冲突（提升立意）
- **核心理念**：冲突双方都有道理，没有绝对对错。
- **设计**：
  - [灰度情境]：如严格防疫vs人道救援
  - [双方理由]：都有合理性
  - [主角选择]：体现成长和格局

### 6. 世界观悬念前置（增加神秘感）
- **核心理念**：不要等到最后才揭秘，前期就要铺垫"不对劲"的细节。
- **设计**：
  - [细节暗示]：如丧尸的异常行为、系统的奇怪提示
```
