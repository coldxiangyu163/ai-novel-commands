### 1.0 ⭐⭐⭐ 重写模式检测（防止字数膨胀）

> **核心问题**：重写章节时，如果读取了原章节内容，AI 会倾向于"保留原有 + 添加新内容"，导致字数膨胀。

**第一步：检测是否为重写操作**

```
# 检查章节文件是否已存在
LS 卷X-卷名/章节/
# 或
Glob: 卷X-卷名/章节/第XX章-*.md
```

**检测结果**：
- 如果目标章节文件**已存在** → 这是**重写操作**
- 如果目标章节文件**不存在** → 这是**新写操作**

**第二步：重写模式特殊处理**

**⚠️ 重写模式铁律（防止字数膨胀）**：

1. **禁止读取本章原文**：
   - ❌ 不要 Read 要重写的章节文件
   - ❌ 不要参考本章的章节摘要内容
   - ✓ 只读取大纲/细纲中本章的要求

2. **前文读取调整**：
   - 新写第N章 → 读取第(N-5)到第(N-1)章
   - 重写第N章 → 读取第(N-5)到第(N-1)章（**跳过第N章本身**）

3. **字数控制强化**：
   - 重写时必须**严格按照 AGENTS.md 的目标字数**
   - 禁止"参考原文长度"
   - 禁止"保留原有内容"的心态

4. **输出提示**：
```markdown
===== 重写模式已激活 =====
⚠️ 检测到第X章已存在，进入重写模式
- 原章节文件：已跳过读取（防止字数膨胀）
- 原章节摘要：已跳过参考
- 字数目标：严格按 AGENTS.md 设定（XXXX字）
- 写作策略：从零开始，只依据大纲/细纲
===========================
```

**为什么重写会字数膨胀？**
- AI 看到原文后，会试图"不丢失原有信息"
- 结果：原内容 + 新增内容 = 字数膨胀
- 解决：重写时完全不看原文，从大纲重新写

### 1.1.3 ⭐ 本章大纲提取（细纲不存在时执行）

> **核心原则**：大纲是硬性约束，不是参考建议。写作必须严格遵循大纲。

**第一步：读取卷大纲**
- 读取 `卷X-卷名/卷X大纲.md` 完整文件
- 定位到本章（第Y章）的大纲内容

**⭐⭐⭐ 章节定位规则（必须严格执行，防止章节错位）**

> **铁律**：必须使用精确匹配，禁止模糊搜索！

**精确匹配模式**：
```
章节标题格式：#### 第{章节号}章：{章节标题}
示例：#### 第44章：黄雀在后
```

**定位步骤**：
1. **计算绝对章节号**：
   - 如果输入的是卷内相对章号，需要转换为绝对章号
   - 公式：绝对章号 = 卷起始章号 + 相对章号 - 1
   - 示例：卷二第14章 = 31 + 14 - 1 = 第44章

2. **精确搜索章节标题**：
   - 搜索模式：`#### 第{绝对章号}章：`（注意冒号是中文冒号）
   - ❌ 错误：搜索 `第44` 或 `44章`（可能匹配到第45章等）
   - ✓ 正确：搜索 `#### 第44章：`（精确匹配）

3. **提取章节内容边界**：
   - **起始边界**：`#### 第{N}章：` 这一行
   - **结束边界**：下一个 `#### 第` 开头的行（不含）或文件结束
   - 提取两个边界之间的所有内容

4. **验证章节标题**：
   - 提取后必须输出完整章节标题进行确认
   - 格式：`第{N}章：{标题}`
   - 如果标题与预期不符，**立即停止并报错**

**错误示例（禁止）**：
```
❌ 搜索"第4"开头 → 可能匹配到第40、41、42...49章
❌ 搜索"章：黄雀" → 可能在其他位置匹配
❌ 不验证标题直接使用 → 可能用错章节内容
```

**正确示例**：
```
✓ 要写第44章 → 搜索"#### 第44章："
✓ 找到后提取标题 → "第44章：黄雀在后"
✓ 验证标题正确 → 继续提取内容
✓ 内容边界 → 从"#### 第44章："到"#### 第45章："之前
```

```markdown
| 3 | [剧情点3] | [场景描述] | ★★☆☆☆ |
...

⚠️ 检查：是否所有大纲剧情点都有对应场景？
- [ ] 剧情点1 → 场景X ✓
- [ ] 剧情点2 → 场景X ✓
- [ ] 剧情点3 → 场景X ✓
```

**第二步：检查场景完整性**

- [ ] 每个大纲剧情点都有对应场景？

- [ ] 场景顺序符合大纲规划？

**如果有 `<beat-sheet>`**：直接使用 `<beat>` 标签定义的内容作为场景。

### 2.2 详略判断标准

| 场景类型 | 详略度 | 字数占比 |
|----------|--------|----------|

| 高潮/打脸/战斗/情感爆发 | ★★★★★ | 35-50% |

### 2.3 过渡检查

检查场景之间是否需要过渡：
- 时间跳跃 → 需要过渡
- 情绪转变 → 需要过渡
- 场景切换 → 视情况

## 阶段 2.5：情绪曲线设计（⭐让读者欲罢不能）

> 好章节不是平铺直叙，而是让读者的情绪跟着起伏

### 2.5.1 本章情绪曲线

根据场景拆分，设计读者的情绪起伏：

```
场景1    场景2    场景3    场景4    场景5
  │        │        │        │        │
  ▼        ▼        ▼        ▼        ▼
 平静 →  好奇 →  紧张 →  爆发 →  余韵
  ▂       ▃       ▅       █       ▄
```

**情绪节点规划**：
| 场景 | 目标情绪 | 实现手法 |
|------|----------|----------|
| 开头 | [好奇/紧张/期待] | [悬念/冲突/反差] |
| 发展 | [升温/压抑/轻松] | [铺垫/危机/调剂] |
| 高潮 | [燃/爽/虐/甜] | [爆发/反转/释放] |
| 收尾 | [满足/悬念/期待] | [余韵/钩子/伏笔] |

### 2.5.2 情绪类型速查

| 情绪 | 触发手法 | 适用场景 |
|------|----------|----------|
| **紧张** | 危机、时限、未知威胁 | 战斗前、逃亡、潜入 |
| **燃** | 逆转、突破、宣言 | 战斗高潮、觉醒 |
| **爽** | 打脸、碾压、揭秘 | 复仇、装逼、反转 |
| **甜** | 互动、暧昧、确认 | 感情戏 |
| **虐** | 失去、误会、牺牲 | 情感低谷 |
| **好奇** | 悬念、伏笔、暗示 | 开头、转折 |
| **轻松** | 日常、吐槽、调侃 | 战后、过渡 |

### 2.5.3 节奏呼吸检查

**避免情绪疲劳**：
- 连续紧张不超过3个场景 → 必须有喘息
- 连续高潮不超过2章 → 需要缓冲章
- 大虐之后 → 必须有甜/爽调剂

**本章节奏**：
- [ ] 有情绪起伏（不是平的）
- [ ] 高潮前有铺垫（不是突然爆发）
- [ ] 高潮后有余韵（不是戛然而止）
- [ ] 与前章情绪有变化（不是连续同一基调）

## 阶段 2.6：悬念与钩子设计（⭐抓住读者）

### 2.6.1 本章悬念布局

**大悬念**（贯穿全章或多章）：
- [悬念内容]：本章是否揭示？ → 是/否/部分

**小悬念**（场景级，推动阅读）：
| 位置 | 悬念内容 | 何时解答 |
|------|----------|----------|
| 第1个场景末 | [小悬念] | 第2个场景 |
| 第2个场景末 | [小悬念] | 第3个场景 |
| 章节末 | [可选：钩子/余韵/自然收束] | 视情况 |

### 2.6.2 结尾方式选择（⭐智能判断）

> **核心原则**：自然过渡优先，钩子是例外而非常态。滥用钩子会导致读者疲劳。

**第一步：判断本章是否需要钩子**

| 章节情况 | 推荐结尾 |
|----------|----------|
| 卷末1-3章 / 重大转折前 | 强钩子 |
| 日常章节 / 过渡章节 | 弱钩子或余韵 |
| 情感高潮后 / 战斗结束后 | 余韵或自然收束 |
| 阶段性完结 | 自然收束 |

**第二步：选择具体方式**

**强钩子类型**（用于关键章节）：
| 类型 | 说明 | 示例 |
|------|------|------|
| **危机型** | 突发危险、敌人出现 | "门外传来脚步声" |
| **悬疑型** | 抛出疑问、发现异常 | "他注意到一个细节" |
| **反转型** | 意外揭示、身份反转 | "来人竟然是..." |
| **期待型** | 预告好事、机遇来临 | "明天，就是突破的日子" |

**弱钩子/余韵类型**（用于普通章节）：
| 类型 | 说明 | 示例 |
|------|------|------|
| **暗示型** | 轻微疑问、小伏笔 | "她似乎还有话没说" |
| **情绪延续** | 氛围收束、情感余韵 | "夜色渐深，她望着窗外" |
| **状态定格** | 人物状态、场景定格 | "他靠在墙上，闭上了眼" |

**自然收束**（用于阶段完结）：
- 事件完整解决，不刻意制造悬念
- 给读者喘息空间

**本章结尾**：[结尾方式] - [具体内容]（可留空表示自然收束）

**⭐⭐⭐ 铁律：反文艺结尾（Anti-Literary Ending）**
> 任何时候，如果你的结尾写出了"文艺腔"，请立即删除重写！

**❌ 严禁使用的结尾类型**：
- "命运的齿轮开始转动。"
- "暴风雨前的宁静。"
- "这场游戏，才刚刚开始。"
- "黑暗中，有一双眼睛注视着一切。"
- "他不知道的是，未来已经改变。"

**✓ 必须使用的结尾类型**：
- **回归日常**："明天还得早起抢单，这破事真烦。"
- **关注当下**："他裹紧了被子。这外卖再不送就凉了。"
- **具体动作**："陈幕翻了个身，把手机静音了。"
- **否定宏大**："至于什么世界末日？不存在的，先吃个煎饼果子再说。"

### 2.6.3 信息差利用

| 信息差类型 | 本章是否使用 | 内容 |
|------------|--------------|------|
| 读者知道、角色不知道 | 是/否 | [内容] |
| 角色知道、读者不知道 | 是/否 | [内容] |
| 部分角色知道 | 是/否 | [内容] |

### 3.0 ⭐ 风格即时约束（写作时强制参照）

1.  **段落碎片化**：

**⭐⭐⭐ 重写模式字数控制（防止膨胀）**：

> **铁律**：重写章节时，字数必须严格控制在目标范围内，禁止参考原章节长度！

## 阶段四：自动强化

### 4.1 强化开头
- 检查开头是否有钩子
- 如果平淡 → 改成悬念/动作/冲突开头

### 4.2 强化爽点
- 找出本章爽点场景
- 确保爽点篇幅足够，情绪释放到位

### 4.3 打磨对话
- 按 AGENTS.md 中定义的主角人设检查
- 按 `写作风格.md` 的对话规范检查（是否需要动作/神态，视风格而定）

### 4.4 增加张力
- 紧张场景参考 `写作风格.md` 的节奏要求
- 增加符合风格定义的感官描写

### 4.5 强化感情戏（如本章有感情线）
- 检查感情互动是否符合当前阶段
- 确保甜蜜/虐心场景情感到位
- 女主人设是否一致
- 对话是否有化学反应

### 4.6 强化结尾（⭐智能判断钩子）

> **核心原则**：自然过渡优先，钩子是例外而非常态。滥用钩子会导致读者疲劳。

**默认使用自然收束**（大多数章节）：
- 事件完整解决，不刻意制造悬念
- 人物状态定格，给读者喘息空间
- 日常章节、过渡章节、情感戏收尾
- 刚经历高潮后的缓冲章
- 阶段性完结章节（如小boss打完）
- 示例："他翻了个身，闭上眼睛。明天还要早起送餐。"

**可以用轻悬念的情况**（少数章节）：
- 下一章有重要剧情需要铺垫
- 本章结尾自然引出疑问
- 示例："她似乎还有话没说。"

**需要强钩子的情况**（仅限关键节点）：
- 卷末最后1-2章（吸引读者追下一卷）
- 重大转折前（如身份揭露、危机爆发）
- 大纲明确标注需要钩子

**本章结尾判断**：
1. **默认选择自然收束**
2. 检查大纲是否明确标注钩子要求
3. 判断章节在卷中的位置（仅卷末考虑强钩子）
4. 选择合适的结尾方式：
   - **自然收束**（默认）：事件完结/状态定格/日常结束
   - **轻悬念**（偶尔）：暗示/疑问/预告
   - **强钩子**（仅关键节点）：悬念/威胁/反转

**⚠️ 禁止文艺腔结尾**：
- ❌ "风雨欲来""命运的齿轮""暴风雨前的宁静"
- ✓ 用具体动作/状态收尾，不用抽象意象

### 4.6.1 名场面检查（如本章有名场面）

**如果本章大纲标注了名场面，检查：**

1. **铺垫是否充分？**
   - 名场面前有足够的情绪积累吗？
   - 读者是否已经期待这一刻？

2. **篇幅是否足够？**
   - 名场面至少500字
   - 不能一笔带过

3. **写法是否到位？**
   - 节奏放慢了吗？（慢镜头感）
   - 有多感官描写吗？（视觉+听觉+触觉）
   - 有金句/燃点吗？

4. **余韵是否保留？**
   - 名场面后不要立刻切换
   - 让情绪在读者心里停留

5. **呼应检查**
   - 是否与前面的伏笔/铺垫形成呼应？
   - 是否让读者有"原来如此"的感觉？


### 4.7 ⭐ 人物变化检查（必须执行）

**确保人物有细微的变化和成长，不是每章都一样：**

1. **主角状态符合吗？**
   - 本章主角情绪是否与 `人物当前状态.md` 一致？
   - 经历了重大事件后，情绪/态度是否有变化？

2. **女主表现有变化吗？**
   - 对主角的态度是否符合当前感情阶段？
   - 表达方式是否与最近几章不同？（参考表达库轮换）
   - 有没有细微的心理变化体现？

3. **配角有存在感吗？**
   - 配角是否沦为背景板/工具人？
   - 本章有配角的闪光点吗？（参考闪光点规划）
   - 配角的表达方式是否与其他角色有区分？

4. **表达方式重复了吗？**
