# 角色声音设计：$ARGUMENTS

> **核心理念**：读者闭上眼睛，只看对话，就能分辨出是谁在说话。这就是角色声音（Voice）的力量。

---

## 为什么需要角色声音？

**问题**：AI写的对话，所有角色说话都像一个人——
- 句式相似
- 用词雷同
- 语气单一
- 缺乏个性

**解决**：为每个角色设计独特的"声音指纹"，让对话有辨识度。

- **情绪表达**：[内敛/外放/压抑/直接]

## 第三步：声音来源分析

> 声音不是凭空设计的，而是来自角色的背景、性格、经历。

```markdown
## 声音来源分析

### 背景影响
- **出身**：[贵族/平民/底层] → 影响用词层次
- **教育**：[高/中/低] → 影响表达复杂度
- **职业**：[军人/商人/学者/...] → 影响专业词汇
- **地域**：[如有方言/口音特点]

### 性格影响
- **内向/外向**：→ 影响话多话少
- **自信/自卑**：→ 影响语气强弱
- **理性/感性**：→ 影响表达方式
- **谨慎/冲动**：→ 影响说话前的思考

### 经历影响
- **创伤(Wound)**：→ 某些话题会[回避/敏感/攻击性]
- **Lie影响**：→ 说话中会体现错误信念
- **关系经历**：→ 对不同人的信任度不同

### 当前状态影响
- **弧光阶段**：→ 随成长，说话方式会微调
- **情绪状态**：→ 情绪激动时声音会变化
- **与对话者关系**：→ 对不同人说话方式不同
```

## 批量模式（all）

当参数为 `all` 时：

1. 读取 `人物档案/` 目录，识别所有主要角色
2. 按重要性排序：主角 → 女主 → 核心配角 → 重要配角
3. 逐个设计声音档案
4. 生成声音对比表
5. 确保声音差异化

---

## 注意事项

1. **声音要有来源**：不要凭空设计，要基于角色背景和性格
2. **差异化优先**：宁可夸张也不要雷同
3. **一致性**：同一角色在不同场景保持声音一致（情绪变化除外）
4. **示例最重要**：抽象描述不如具体示例有用
5. **随弧光演变**：声音会随角色成长微调，但核心特征保持
6. **写作时参考**：声音档案要在写作时实际使用才有价值
