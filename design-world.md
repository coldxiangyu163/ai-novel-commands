# 世界观深度设计

**在 init-novel 生成基础框架后，深入设计世界观体系。**

> **重要原则**：以下所有题材模板都是**参考示例**，不是必须遵循的规范。
> 
> - 根据本作品需要**自由选用、修改、删除**模板内容
> - 鼓励**创新和扩展**，不要被模板限制想象力
> - 独特设定请在"自定义设定区"中详细说明
> - 混合题材时，**灵活组合**各题材的世界观元素

---

## 第一步：读取项目文件

1. `AGENTS.md` - 获取题材设定（主题材、副题材、混合比例）
2. `小说规划.md` - 获取已有的世界观概要
3. `自定义设定.md` - 获取本作品独特设定（如存在）

---

## 第二步：根据题材进行深度设计

### 通用世界观要素

无论什么题材，都需要设计：

#### 2.1 时间设定
- 故事发生的时代/年份
- 重要历史节点
- 时间跨度

#### 2.2 空间设定
- 主要地点/区域
- 地理特征
- 重要场所

#### 2.3 社会结构
- 权力层级
- 社会阶层
- 组织/势力分布

#### 2.4 规则体系
- 这个世界的特殊规则
- 能力/力量体系
- 资源/经济体系

---


## 第三步：题材专项设计

> **注意**：不提供固定模板，大模型根据用户的具体题材和设定动态生成。

根据 AGENTS.md 中的题材设定，设计对应的世界观体系。

### 3.1 设计原则

1. **问用户，不要假设**：不清楚的设定要询问用户意见
2. **服务剧情**：世界观是为剧情服务的，不是炫设定
3. **留有余地**：不要把所有细节都定死，为后续扩展留空间
4. **差异化**：参考 AGENTS.md 中的差异化定位，确保世界观体现独特性

### 3.2 必须设计的核心体系

根据题材类型，至少设计以下体系：

| 题材类型 | 必须设计的体系 |
|----------|----------------|
| 玄幻/仙侠 | 修炼境界、功法等级、宗门势力、天材地宝 |
| 末世 | 灾变设定、丧尸/怪物等级、觉醒者体系、势力分布 |
| 都市 | 社会背景、能力体系（如有）、势力/组织 |
| 科幻 | 科技等级、星际势力、种族设定 |
| 历史/权谋 | 朝代设定、官制体系、世家门阀 |
| 游戏/无限流 | 系统规则、副本/任务机制、成长体系 |
| 系统流 | 系统面板、升级机制、任务奖励 |

### 3.3 设计流程

对于每个核心体系，按以下流程设计：

```markdown
## [体系名称]

### 基本框架
- 这个体系的核心规则是什么？
- 有哪些等级/层次划分？
- 如何升级/进阶？

### 与主角的关系
- 主角在这个体系中的起点？
- 主角的成长路径？
- 主角的独特优势/金手指如何融入？

### 与剧情的关系
- 这个体系如何产生冲突？
- 这个体系如何产生爽点？
- 各卷主角在这个体系中的位置？

### 独特设定
- 与同类作品的差异？
- 本作品的创新点？
```


### 4.2 创新元素

```markdown

[完全原创的体系]

### 独创概念/名词
[本作品独有的概念和术语]

[标准模板之外的势力设定]
```

```
├── [核心体系].md       # 如：修炼体系.md、丧尸体系.md

```

1. **动态生成**：根据用户的具体题材和设定生成，不套用固定模板
2. **与约束联动**：世界观设计要能支撑后续的剧情约束
3. **留有余地**：不要把所有细节都定死，为后续扩展留空间
4. **服务剧情**：世界观是为剧情服务的，不是炫设定
5. **体现差异化**：参考 AGENTS.md 中的差异化定位
