# 批量写作任务：$ARGUMENTS

**连续写作多个章节，通过"章节摘要链"保持前后连贯，避免剧情跳跃和细节丢失。**

---

## 核心机制：章节摘要链

为解决批量写作的连贯性问题，采用"摘要链"机制：

```
第1章 → 生成摘要1 → 第2章（读摘要1）→ 生成摘要2 → 第3章（读摘要1+2）→ ...
```

每章摘要包含（约200字）：
- **主要事件**：本章发生了什么
- **关键对话**：重要的对话内容
- **人物状态**：主角/配角的状态变化
- **新增细节**：新出现的人名、地名、物品
- **未完成事项**：待续的事件、悬念
- **伏笔**：埋下或回收的伏笔
- **感情线进展**：感情状态变化（如有感情线）

---

## 执行流程

### 第一步：读取基础文档（只读一次）

1. `AGENTS.md` - 本书写作规范
2. `写作灵感库.md` - 灵感参考
3. `写作风格.md` - 风格设定（（如果存在））
4. `小说规划.md` - 总体规划
5. 对应卷的完整大纲
6. `人物档案/` - 主角和相关人物档案
7. 已有章节的摘要（如果存在 `卷X-卷名/章节摘要.md`）

### 第二步：初始化摘要链

如果是从第1章开始：创建空的摘要链
如果是续写：读取已有的章节摘要

### 第三步：循环写作每一章

```
对于每一章：
│
├─ 1. 读取上下文
│     - 前一章完整内容（最近1章）
│     - 前面所有章节的摘要链
│     - **定位本章大纲**：从卷大纲文件中提取第X章的具体规划（关键剧情、出场人物、爽点）
│
├─ 2. 写作前检查
│     - 确认要承接的剧情点
│     - 确认要延续的细节
│     - 确认本章要推进的事项
│
├─ 3. 写初稿（5000字）
│
├─ 4. 自动强化
│     - 开头承接前章
│     - 爽点到位
│     - 对话打磨
│     - 结尾钩子
│
├─ 5. 连贯性自检
│     - 是否有剧情跳跃？
│     - 是否遗漏了前章的待续事项？
│     - 细节是否一致？
│
├─ 6. 生成本章摘要（200字）
│     追加到摘要链
│
├─ 7. 保存章节
│
└─ 8. 继续下一章
```

### 第四步：输出格式

每章输出：
```
===== 第X章《标题》=====

[章节正文]

---
字数：XXXX
爽点：XXX
承接：[承接了前章的什么]
伏笔：[埋设/回收]
===== 第X章 结束 =====

【本章摘要】
主要事件：...
关键细节：...
待续事项：...
```

### 第五步：批量写作完成后

1. 保存完整的摘要链到 `卷X-卷名/章节摘要.md`
2. 输出总结报告

```
===== 批量写作完成 =====

完成章节：第X章 - 第X章
总字数：XXXXX字

【连贯性报告】
✓ 剧情线：[主线推进情况]
✓ 人物线：[人物状态变化]
✓ 伏笔线：[埋设X个，回收X个]

【待续事项】（下次续写需注意）
1. ...
2. ...

【章节摘要已保存】
→ 卷X-卷名/章节摘要.md

=============================
```

## 章节摘要模板

每章写完后生成：

```markdown
### 第X章《标题》摘要

**主要事件**：
- [事件1]
- [事件2]

**关键细节**：
- 人物：[新出场/状态变化]
- 地点：[新地点/位置变化]
- 物品：[获得/使用/丢失]
- 数据：[等级/数量变化]

**重要对话**：
- 「...关键台词...」

**待续事项**：
- [未完成的事]

**伏笔**：
- 埋设：[...]
- 回收：[...]

**感情线**（如有）：
- 本章感情状态：[相遇/相识/心动/确定/考验]
- 女主出场：是/否
- 感情事件：[简述]
- 关系变化：[无变化/升温/冷却/冲突]
```

## 质量要求

每章必须满足：
- [ ] 字数达标
- [ ] 开头承接前章（不能跳跃）
- [ ] 爽点到位
- [ ] 主角符合人设
- [ ] 细节与前文一致
- [ ] 结尾有悬念
- [ ] 生成章节摘要
- [ ] 感情线符合规划（如有）
- [ ] **像高手一样写**（留白、展示、简洁）
- [ ] **角色有独特表达**（不是都在点头）
- [ ] **详略得当**（高潮充分展开，日常快速带过）
- [ ] **过渡自然**（场景切换不生硬）

---

## 建议

1. **单次不超过5章**：超过5章建议分批，中间做一次一致性检查
2. **及时保存摘要**：摘要链是续写的关键
3. **重要章节单独写**：大爽点/大转折章节建议用 /auto-write 精雕细琢

---

## 注意事项

1. **摘要链是核心**：这是保持连贯的关键机制
2. **不要跳过摘要生成**：每章必须生成摘要
3. **细节追踪**：特别注意人名、地名、数字的一致性
4. **承上启下**：每章开头必须自然承接前章结尾
