# 承诺系统分析：$ARGUMENTS

**读者追读的本质是"等待承诺兑现"。本命令系统化管理故事中的所有承诺。**

> **核心理念**：
> - 故事是一系列"承诺"的设立与兑现
> - 承诺太久不兑现，读者会失去耐心
> - 承诺兑现不够精彩，读者会失望
> - 好的承诺管理 = 持续的期待感 + 满意的兑现感

## 第二步：理解承诺系统

### 2.1 什么是"承诺"？

**承诺（Promise）** = 作者向读者发出的"期待信号"

当你写下某些内容时，就是在向读者承诺"这个会有后续/回报"：
- 提出一个问题 → 承诺会给答案
- 制造一个冲突 → 承诺会有解决
- 引入一个角色 → 承诺他会有作用
- 埋下一个伏笔 → 承诺会有回收
- 建立一个期待 → 承诺会有满足

### 2.2 承诺的生命周期

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   设立      │ →  │   维护      │ →  │   兑现      │
│ (Promise)   │    │ (Progress)  │    │ (Payoff)    │
└─────────────┘    └─────────────┘    └─────────────┘
     ↓                   ↓                   ↓
  抛出期待           定期喂料            回报满足
  读者好奇           保持期待            读者满意
```

### 2.3 承诺的类型

| 类型 | 说明 | 周期 | 示例 |
|------|------|------|------|
| **类型承诺** | 类型带来的基本期待 | 全书 | 末世文→生存刺激 |
| **主线承诺** | 故事核心目标 | 全书 | 主角会复仇成功 |
| **悬念承诺** | 提出的谜题/问题 | 长/中/短 | 系统的来源是什么？ |
| **角色承诺** | 角色相关的期待 | 中/短 | 女主身世之谜 |
| **关系承诺** | 关系发展期待 | 长/中 | 他们会在一起吗？ |
| **冲突承诺** | 冲突解决期待 | 中/短 | 主角vs仇人的清算 |
| **能力承诺** | 能力展示期待 | 中/短 | 新技能什么时候用？ |

## 注意事项

1. **承诺必须兑现**：设立的承诺必须兑现，否则就是欺骗读者
2. **不能透支**：同时进行的承诺不能太多，读者记不住
3. **定期喂料**：长线承诺必须定期提及，防止遗忘
4. **兑现要精彩**：等待越久的承诺，兑现要越精彩
5. **宁缺毋滥**：不确定能好好兑现的承诺，不要轻易设立
