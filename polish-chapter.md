## 第三步：详略检查与调整（核心步骤）

> **⚠️ 这是最重要的步骤！精简之前必须先判断场景类型，高潮场景绝对不能精简！**

### 3.1 识别高潮场景（必须先做）

**高潮场景识别标准**（符合任意一条即为高潮场景）：

| 类型 | 识别标准 | 示例 |
|------|----------|------|
| **重大转折** | 主角命运/状态发生根本改变 | 重生醒来、觉醒异能、身份揭露 |
| **情感爆发** | 强烈情感释放的时刻 | 复仇、告白、生离死别、仇恨宣泄 |
| **战斗高潮** | 关键战斗的决胜时刻 | Boss战、以弱胜强、绝地反击 |
| **打脸爽点** | 让读者爽的核心场景 | 装逼打脸、反转揭秘、碾压对手 |
| **悬念揭晓** | 重大悬念的揭开 | 身世真相、阴谋揭露、谜底揭晓 |

**本章高潮场景清单**：
```
□ 场景1：[描述] → ★★★★★ 必须详写
□ 场景2：[描述] → ★★★★★ 必须详写
...
```

### 3.2 识别日常场景（可以精简）

**日常场景识别标准**：

| 类型 | 识别标准 | 处理 |
|------|----------|------|
| 赶路/移动 | 从A到B的过程 | 一句话带过 |
| 日常对话 | 无剧情推动的闲聊 | 大幅精简 |
| 信息交代 | 单纯说明性内容 | 融入行动 |
| 过渡衔接 | 场景切换 | 简洁过渡 |

### 3.3 场景详略分析表

```markdown
## 本章场景详略分析

| 场景 | 类型判断 | 当前状态 | 操作 |
|------|----------|----------|------|
| [场景1] | 高潮-重大转折 | 过于简略 | 🔴 扩写 |
| [场景2] | 高潮-情感爆发 | 详细但有AI味 | 🔵 优化表达 |
| [场景3] | 高潮-打脸 | 已经很好 | 🟢 保持 |
| [场景4] | 日常-对话 | 冗余拖沓 | 🟡 精简（可选）|
| [场景5] | 日常-过渡 | 有AI味 | 🔵 优化表达 |
| [场景6] | 铺垫 | 合适 | 🟢 保持 |

操作说明：
🔴 扩写 - 高潮场景不够展开，需要增加内容
🟡 精简 - 日常场景确实冗余（可选，不强制）
🟢 保持 - 已经合适，不改动
🔵 优化表达 - 有AI味但字数合适，只改表达方式
```

### 3.4 高潮场景保护机制

**🔴 以下场景禁止精简，只能保持或扩写**：

1. **开篇钩子**：第一章的开头，必须抓人
2. **重生/穿越/觉醒时刻**：主角状态改变的关键时刻
3. **仇恨建立场景**：让读者理解主角恨意的场景
4. **复仇/打脸场景**：核心爽点
5. **情感高潮**：告白、诀别、重逢等
6. **每章的核心爽点**：本章最爽的那个场景

**判断问题**：
- 这个场景删掉/精简后，读者会不会觉得"不过瘾"？
- 这个场景是不是本章存在的意义？
- 读者期待看到这个场景的详细展开吗？

**如果答案是"是"，则禁止精简。**

### 3.5 详略调整建议

**🔴 需要扩写的场景**（高潮不够展开）：
- 场景X：当前X字，建议扩写到X字
- 扩写方向：[增加什么内容——节奏、细节、情感、氛围]

**🟡 需要精简的场景**（日常太拖沓）：
- 场景X：当前X字，建议精简到X字
- 精简方向：[删除什么内容]

**🔵 需要补充过渡**：
- 位置：[哪两个场景之间]
- 过渡内容：[简要说明]

### 3.6 字数分配原则

- 高潮场景：**至少占总字数的50%**
- 铺垫场景：约25%
- 日常/过渡：**不超过25%**

**如果高潮场景占比不足50%，优先扩写高潮，而非精简日常。**

## 第四步：执行风格化润色

根据 `写作风格.md` 定义的原则逐段重写。

**示例（仅供参考，具体请按风格文件执行）**：

如果风格是 **"简洁留白"**：
> 原文：唐健的目光扫过众人。"出发。"他说。
> 改后："出发。"

如果风格是 **"心理流/文青"**：
> 原文：唐健的目光扫过众人。"出发。"他说。
> 改后：唐健看着眼前这些熟悉又陌生的脸孔，心里的石头沉了沉。最终，他只吐出两个字："出发。"

**核心要求**：
1. **严格遵循风格定义**：不要用通用标准去套，要用书的特定风格去套。
2. **保持信息量**：改表达方式，不删减核心剧情。
3. **场景适配**：根据 2.5 中的详略偏好处理不同场景。

## 第五步：执行润色

### 5.1 根据场景类型决定操作

**四种操作类型**：

| 操作 | 适用情况 | 说明 |
|------|----------|------|
| 🔴 **扩写** | 高潮场景不够展开 | 增加节奏、细节、情感 |
| 🟡 **精简** | 日常场景确实冗余 | **可选**，不强制 |
| 🟢 **保持** | 已经合适 | 不改动 |
| 🔵 **优化表达** | 有AI味但字数合适 | 只改表达，不改字数 |

**执行顺序**：
1. **先处理高潮场景**（扩写或优化表达）
2. **再处理日常场景**（精简、优化表达或保持）
3. **最后补充过渡**

**⚠️ 重要：精简不是必须的！**
- 如果日常场景写得不错，只需优化表达去AI味
- 只有确实冗余拖沓的才需要精简
- 不要为了精简而精简

### 5.2 扩写高潮场景（按需执行）

**先判断是否需要扩写**：

| 情况 | 判断 | 操作 |
|------|------|------|
| 高潮场景已经很详细 | 节奏感好、画面感强、读者能享受 | ✅ 不需要扩写 |
| 高潮场景过于简略 | 一笔带过、缺乏节奏、不够爽 | ⚠️ 需要扩写 |

**判断标准**：
- 高潮场景字数占比是否达到40%以上？
- 是否有节奏控制（短句、停顿、慢镜头）？
- 是否有足够的画面感和爽感？
- 读者读完会不会觉得"不过瘾"？

**如果需要扩写，扩写方向**：
- 增加节奏控制（短句、停顿、慢镜头）
- 增加感官细节（视觉、听觉、触觉）
- 增加角色反应（旁观者的震惊、对手的恐惧）
- 增加氛围渲染（环境、气氛）

**扩写示例**：
```
原文（过于简略，需要扩写）：
唐健杀了周虎。

扩写后：
唐健没有说话。
周虎被这沉默盯得后背发凉。
"你、你想干什么？"他的声音开始发颤。
三秒。
唐健动了。
刀光一闪。
周虎的眼睛瞪得老大，他甚至没看清那一刀是怎么出的。
血溅三尺。
尸体倒下的声音，在寂静的夜里格外清晰。
```

**不需要扩写的情况**：
```
原文（已经很详细，保持原样）：
唐健没有说话，只是看着周虎。
周虎被这眼神盯得后背发凉，嘴里的狠话突然说不出来了。
空气仿佛凝固了。
三秒后，唐健动了。
刀光闪过，快得让人看不清轨迹。
血溅三尺。
周虎的身体软软地倒下，眼睛还瞪得老大。

→ 已经有节奏感和画面感，不需要扩写。
```

### 5.2.1 不同类型高潮场景的扩写方向

**重生/穿越/觉醒场景**：
```
过于简略：
唐健猛地坐起，冷汗浸透后背。他重生了。

应该有的内容：
- 噩梦/死亡的余韵（身体记忆、恐惧残留）
- 确认自己还活着的过程（触摸、呼吸、心跳）
- 记忆与现实的冲突（这是哪里？什么时候？）
- 情感的涌动（震惊、庆幸、愤怒、决心）
- 对环境的重新感知（熟悉又陌生）
```

**仇恨/复仇宣言场景**：
```
过于简略：
他写下七个名字，决定这一世一个都不放过。

应该有的内容：
- 每个仇人背后的记忆闪回（哪怕一句话）
- 写下名字时的情感（恨意、冷静、决绝）
- 具体的复仇计划或态度
- 读者能感受到的"爽感预期"
```

**战斗/打脸场景**：
```
过于简略：
唐健一招制敌，对方倒地不起。

应该有的内容：
- 战斗前的气氛铺垫（对峙、压迫感）
- 动作的节奏感（快慢结合、短句）
- 对手/旁观者的反应（震惊、恐惧）
- 战斗后的余韵（沉默、敬畏）
```

**情感高潮场景**：
```
过于简略：
她看着他，眼眶红了。"你回来了。"

应该有的内容：
- 情感爆发前的铺垫（压抑、克制）
- 细节动作（颤抖、哽咽、下意识的小动作）
- 环境与情感的呼应
- 留白的艺术（不说破，让读者自己感受）
```

### 5.3 精简日常场景（可选，不强制）

**先判断是否需要精简**：

| 情况 | 判断 | 操作 |
|------|------|------|
| 日常场景确实冗余 | 拖沓、啰嗦、信息密度低 | 🟡 精简 |
| 日常场景写得不错 | 节奏合适、有信息量 | 🟢 保持或🔵优化表达 |

**什么情况才需要精简**：
- 同一个意思翻来覆去说
- 大段无用的环境描写
- 对话中有太多废话
- 明显拖节奏的段落

**什么情况不需要精简**：
- 字数不多但信息完整
- 节奏感已经不错
- 只是有些AI味表达 → 用"优化表达"而非"精简"

### 5.4 优化表达（去AI味但不改字数）

**适用情况**：场景字数合适，但有AI味表达需要优化

**操作方式**：
- 用高手原则替换AI味句式
- 保持原有的信息量和节奏
- 只改表达方式，不删减内容

**示例**：
```
原文（有AI味）：
他的目光中闪过一丝复杂的神色，似乎在思考着什么。

优化后（去AI味，字数相近）：
他没说话，手指在桌上敲了两下。

→ 信息量相同（都是"在思考"），但表达方式更高手。
```

### 5.5 补充过渡

对第三步识别出的"需要过渡"的位置，补充自然过渡。

### 5.6 保持原则

- ✅ 可以改：表达方式、用词、句式
- ✅ 可以扩写：高潮场景不够展开时
- ✅ 可以精简：日常场景确实冗余时（可选）
- ✅ 可以优化表达：有AI味但字数合适时
- ❌ 不能改：剧情走向、人物行为、关键信息
- ⚠️ 谨慎改：对话内容（只改标记，不改台词）

## 附录：常见问题表达替换（参考）

> **注意**：以下仅为通用建议，请优先参考 `写作风格.md` 中的 **"禁忌清单"** 和 **"用词规范"**。如果风格文件中允许某些表达，则忽略下表。

| 问题表达 | 处理方式 | 替换建议 |
|----------|----------|----------|
| 目光扫过 | 视风格而定 | 通常可删除，或改为心理描写 |
| 眼中闪过一丝X | 视风格而定 | 动作流风格建议换成动作 |
| 嘴角微微上扬 | 视风格而定 | "笑了"或删除，避免油腻 |
| 不禁/不由得 | 建议删除 | 直接写动作/心理 |
| 心中暗道 | 建议删除 | 直接写想法 |
| 竟然/居然 | 建议删除 | 用反应表达惊讶 |
