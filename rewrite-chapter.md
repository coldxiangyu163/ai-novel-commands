# 章节重写：$ARGUMENTS

> **定位**：根据check-*命令的检查结果，针对性重写问题章节。支持全部重写或局部修改。

---

## 第一步：读取原章节和检查报告

### 1.1 原章节
1. `卷X-卷名/章节/第X章-[标题].md` - 原章节正文
2. `卷X-卷名/章节/第X章-细纲.md` - 章节细纲（如存在）

### 1.2 检查报告
3. `质量检查/` 目录下的相关检查报告
4. 或用户直接提供的问题描述

### 1.3 参考文件
5. `写作风格.md` - 风格规范
6. `AGENTS.md` - 人设铁律
7. 相关人物档案
8. `动态状态/人物当前状态.md`
9. 前后章节（确保衔接）

---

## 第二步：问题诊断

### 2.1 问题分类

```markdown
## 问题诊断报告

### 检查来源
- [ ] /check-logic - 逻辑检查
- [ ] /check-consistency - 一致性检查
- [ ] /check-structure - 结构检查
- [ ] /check-volume - 卷检查
- [ ] /review-reader - 读者视角评审
- [ ] 用户反馈
- [ ] 其他：[来源]

### 问题列表
| # | 问题类型 | 问题描述 | 位置 | 严重程度 |
|---|----------|----------|------|----------|
| 1 | [类型] | [描述] | [段落/场景] | 严重/中等/轻微 |
| 2 | [类型] | [描述] | [段落/场景] | 严重/中等/轻微 |
| ... | ... | ... | ... | ... |

### 问题类型说明
| 类型 | 说明 | 修改范围 |
|------|------|----------|
| 逻辑硬伤 | 时间/空间/因果逻辑错误 | 局部修改 |
| 人设崩塌 | 人物言行不符合设定 | 局部修改 |
| 节奏问题 | 太快/太慢/拖沓 | 可能需要大改 |
| 情节问题 | 剧情不合理/强行推进 | 可能需要大改 |
| 伏笔问题 | 伏笔遗漏/错误 | 局部修改 |
| 衔接问题 | 与前后章节不衔接 | 开头/结尾修改 |
| 风格问题 | 文风不一致/描写问题 | 润色修改 |
| 对话问题 | 对话不自然/不符人设 | 局部修改 |
| 弧光问题 | 人物弧光体现不当 | 可能需要大改 |
```

### 2.2 修改策略判断

```markdown
## 修改策略

### 问题严重程度评估
- 严重问题数量：X个
- 中等问题数量：X个
- 轻微问题数量：X个

### 修改策略选择

#### 策略A：局部修改
**适用条件**：
- 问题集中在特定段落/场景
- 不涉及整体结构调整
- 严重问题≤2个

**执行方式**：
- 定位问题段落
- 针对性修改
- 检查前后衔接

#### 策略B：场景重写
**适用条件**：
- 某个场景整体有问题
- 需要重新设计场景内容
- 但不影响其他场景

**执行方式**：
- 重写问题场景
- 调整与其他场景的过渡
- 检查整体连贯性

#### 策略C：全章重写
**适用条件**：
- 问题涉及整体结构
- 严重问题>3个
- 节奏/情节需要大幅调整

**执行方式**：
- 参考原细纲（或重新设计）
- 完全重写
- 相当于重新执行 /write-chapter

### 本次选择：[策略A/B/C]
理由：[为什么选择这个策略]
```

## 常见问题修改指南

### 逻辑问题修改

| 问题类型 | 修改方法 |
|----------|----------|
| 时间矛盾 | 调整时间描述，确保与前后文一致 |
| 空间跳跃 | 补充位置转移的描写 |
| 因果错误 | 调整事件顺序或补充因果链 |
| 能力超限 | 降低表现或补充获得能力的铺垫 |

### 人设问题修改

| 问题类型 | 修改方法 |
|----------|----------|
| 对话不符 | 参考人物档案的说话风格重写对话 |
| 行为不符 | 参考行为模式调整行动 |
| 性格突变 | 补充心理转变的铺垫或删除突变 |
| 铁律违反 | 必须修改，不能违反人设铁律 |

### 节奏问题修改

| 问题类型 | 修改方法 |
|----------|----------|
| 太快 | 增加细节描写、心理活动、对话 |
| 太慢 | 精简描写、合并场景、删除废话 |
| 拖沓 | 删除无关内容、压缩过渡 |
| 平淡 | 增加冲突、增加情绪波动 |

### 衔接问题修改

| 问题类型 | 修改方法 |
|----------|----------|
| 开头断裂 | 补充与上章的承接 |
| 结尾悬空 | 增加钩子或情绪落点 |
| 场景跳跃 | 补充过渡段落 |

---

## 注意事项

1. **先诊断后修改**：不要急于修改，先完整分析所有问题
2. **选择正确策略**：小问题不要大改，大问题不要小修
3. **保持一致性**：修改后要检查与前后章节的一致性
4. **避免过度修改**：解决问题即可，不要改变原有的优点
5. **记录修改**：重要修改要记录，方便回溯
6. **级联影响**：注意修改可能对其他章节的影响
