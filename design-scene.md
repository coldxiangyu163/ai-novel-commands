# 场景深度设计：$ARGUMENTS

**用于将一个模糊的构思，打磨成充满张力和画面感的精彩场景。**

> **痛点解决**：
> - 觉得大纲"太平"、"没劲"
> - 关键高潮章不知道怎么写才出彩
> - 想不出"意料之外"的情节

---

## 第零步：读取上下文（⭐确保承接 + 避免重复）

**如果用户提供了[卷号]和[章号]参数，必须先读取完整上下文：**

### 0.1 读取大纲与章节上下文

1. **读取卷大纲**：`卷X-卷名/卷X大纲.md`
2. **定位目标章节**：找到第X章的原设计
3. **读取前1-2章**：提取前面章节的：
   - 章末状态（人物在哪、情绪如何、发生了什么）
   - 章末钩子（留下了什么悬念/伏笔）
   - 情绪走向（前章是紧张/轻松/高潮/低谷）
4. **读取后1章**（如有）：了解原本要承接什么

### 0.2 读取全书记忆（⭐避免重复）

| 文件 | 提取内容 |
|------|----------|
| `全书记忆/已用套路库.md` | 近期用过的套路、建议避免的类型 |
| `全书记忆/伏笔追踪表.md` | 本章应回收的伏笔、可埋设的新伏笔方向 |
| `全书记忆/感情线节点.md` | 当前感情阶段、近期互动类型 |

**提取套路约束**：
```markdown
## 套路约束

### 近期已用（建议避免）
- 开场方式：[X]（第Y章刚用）
- 打脸套路：[X]（连续用了Z次）
- 感情名场面：[X]（第Y章刚用）

### 建议尝试（长期未用）
- 开场方式：[X]（已N章未用）
- 打脸套路：[X]（全书未用）
- 感情名场面：[X]（已N章未用）
```

**提取伏笔约束**：
```markdown
## 伏笔约束

### 本章应回收的伏笔
- [伏笔A]：埋设于第X章，原计划本章回收
- [伏笔B]：已拖延Z章，建议本章回收

### 可埋设的新伏笔方向
- 为卷X的[事件]做铺垫
- 为[人物]的后续发展埋线
```

### 0.3 读取动态状态与人物档案

| 文件 | 提取内容 |
|------|----------|
| `动态状态/人物当前状态.md` | 主要人物当前情绪、近期经历 |
| `动态状态/叙事手法记录.md` | 最近章节的叙事手法（避免重复） |
| `人物档案/[相关人物].md` | 本章出场人物的性格、说话方式 |
| `写作风格.md` | 风格定位、禁忌清单 |

**提取人物状态**：
```markdown
## 人物当前状态

### [主角]
- 情绪：[当前情绪]
- 近期经历：[摘要]
- 待处理事项：[未解决的问题]

### [本章相关人物]
- 与主角关系：[当前关系]
- 情绪状态：[当前情绪]
```

### 0.4 提取承接约束

```markdown
## 承接约束（本章设计必须遵守）

### 来自前章的约束
- **前章结束状态**：[人物位置、情绪、状态]
- **前章钩子**：[需要回应的悬念/问题]
- **情绪承接**：前章是[X]，本章开头应该[延续/反差/过渡]

### 来自后章的约束（如需保持后续大纲不变）
- **需要为后章铺垫**：[后章需要的前置条件]
- **不能破坏**：[后章依赖的设定/状态]

### 本章在全卷中的位置
- **节奏定位**：[铺垫/升温/小高潮/大高潮/缓冲]
- **详略度**：[★☆☆☆☆ ~ ★★★★★]
```

### 0.5 设计边界确认

**在开始创意发散之前，先确认：**

| 问题 | 回答 |
|------|------|
| 本章开头必须承接什么？ | [前章钩子/状态] |
| 本章结尾必须留下什么？ | [为后章铺垫/保持悬念] |
| 哪些套路要避免？ | [近期已用的] |
| 哪些伏笔要回收？ | [应回收列表] |
| 人物当前什么状态？ | [情绪/近期经历] |
| 感情线当前什么阶段？ | [阶段/近期互动] |
| 哪些元素可以自由发挥？ | [中间过程/具体方式] |

## 第一步：破除套路（Brainstorming）

针对输入的场景构思，进行三个维度的发散：

### 1. 拒绝第一反应
> 你的第一个想法通常也是读者的第一个想法（俗称套路）。

- **惯性思路**：[分析最容易想到的写法] → ❌ **放弃**
- **反套路A（逆向）**：[完全相反的处理方式]
- **反套路B（侧面）**：[侧面/间接/转换视角的处理方式]

### 2. 增加变量（X-Factor）
> 引入外部干扰，让场景不稳定。

- **环境变量**：[暴雨/狭窄空间/噪音/停电] 如何增加难度？
- **限制条件**：如果主角[受伤/限时/不能说话/必须保护他人]，场景会怎样？
- **意外因素**：突然闯入的第三方？装备故障？情报错误？

### 3. 核心张力提炼
- **表层冲突**：[看得见的动作/语言对抗]
- **深层博弈**：[心理/信息/价值观的对抗]
- **悬念钩子**：读者此刻最想知道什么？

---

## 第二步：画面与氛围（Cinematography）

### 1. 视觉风格
- **光影**：[明暗对比/色调/光源位置]
- **构图**：[压抑/空旷/混乱/对角线]
- **关键意象**：[选一个核心物品/符号贯穿全场]

### 2. 感官沉浸（五感法）
> 至少选择2个非视觉感官进行强化

- **听觉**：[背景音/心跳/突兀的声响/耳鸣]
- **触觉/温度**：[湿黏/燥热/刺痛/冰冷]
- **嗅觉**：[铁锈味/霉味/香水味/血腥味]

---

## 第三步：节奏与动态（Pacing）

### 1. 动态曲线
- **切入点**：[动/静/悬念/特写]（拒绝流水账开头）
- **升温点**：[冲突升级/危机加重]
- **爆发点**：[最高潮的一瞬]
- **落点**：[结局时的余韵/留白]

### 2. 情绪反差
- **起始情绪**：[如：自信满满]
- **结束情绪**：[如：惊恐万状]
- **反差设计**：[如何实现这个剧烈的转变？]

## 第五步：自动注入（可选）

**如果用户提供了[卷号]和[章号]参数：**

1.  **定位文件**：读取 `卷X-卷名/卷X大纲.md`。
2.  **定位章节**：找到第X章的位置。
3.  **注入标签**：在章节描述下方，插入以下XML代码块：

```xml
<scene-design>
  <core-concept>[核心创意]</core-concept>
  <beat-sheet>
    <beat type="切入">[切入点内容]</beat>
    <beat type="铺垫">[铺垫内容]</beat>
    <beat type="转折">[转折内容]</beat>
    <beat type="高潮">[高潮内容]</beat>
    <beat type="收尾">[收尾内容]</beat>
  </beat-sheet>
  <atmosphere>
    <visual>[视觉风格]</visual>
    <sensory>[感官细节]</sensory>
  </atmosphere>
  <instruction>
    <focus>[重点详写]</focus>
    <avoid>[避免写法]</avoid>
  </instruction>
</scene-design>
```

4.  **输出提示**：
    "✅ 场景设计已注入到 [卷X大纲.md]。执行 /auto-write 时将自动应用此方案。"

**如果未提供参数**：
仅输出设计方案，并提示："如需自动保存，请使用格式 `/design-scene <场景> <卷号> <章号>`"

## 第五点五步：大纲标记更新（⭐只更新大纲，不写全书记忆）

> **设计原则**：大纲 = 计划，全书记忆 = 执行记录
> 
> design-scene 只负责在大纲中做标记，auto-write 写完后统一同步到全书记忆

### 5.5.1 检测设计中的关键元素

扫描场景设计，识别以下元素：

| 元素类型 | 检测内容 |
|----------|----------|
| 伏笔 | 暗示、悬念、需要后续解答的问题 |
| 套路/名场面 | 打脸方式、感情名场面、战斗套路 |
| 感情进展 | 感情阶段变化、关键互动 |
| 人物状态 | 重大情绪/状态变化 |

### 5.5.2 更新大纲伏笔清单

**只更新 `卷X大纲.md` 末尾的伏笔清单（不写全书记忆）：**

```markdown
### 本卷伏笔清单

#### 本卷埋设的伏笔
| 伏笔内容 | 埋设章节 | 预计回收 | 来源 |
|----------|----------|----------|------|
| [伏笔描述] | 第X章 | 第Y章/卷Z | plan-volume |
| [新伏笔] | 第X章 | 第Y章 | 📝 design-scene | ← 新增标记

#### 本卷回收的伏笔
| 伏笔内容 | 原埋设位置 | 回收章节 | 来源 |
|----------|------------|----------|------|
| [伏笔描述] | 第X章 | 第Y章 | plan-volume |
| [待回收] | 第X章 | 第Y章 | 📝 design-scene | ← 计划回收
```

### 5.5.3 更新大纲章节标记

在对应章节的设计中，添加伏笔/套路标记：

```markdown
#### 第X章：[标题]

**【伏笔标记】**（📝 design-scene 新增）
- 埋设：[伏笔内容] → 预计第Y章回收
- 回收：[伏笔内容] → 来自第Z章

**【套路标记】**（📝 design-scene 新增）
- 打脸方式：[类型]
- 名场面：[类型]

**【感情线标记】**（📝 design-scene 新增）
- 互动类型：[类型]
- 阶段进展：[从X到Y]
```

### 5.5.4 伏笔回收提醒

**读取大纲伏笔清单，检测本章应回收的伏笔：**

```
📌 伏笔回收提醒

根据大纲伏笔清单，以下伏笔应在本章回收：
- [伏笔A]：埋设于第X章，原计划本章回收
- [伏笔B]：埋设于第Y章，已拖延Z章

请在场景设计中体现回收，或在大纲中调整回收时机。
```

### 5.5.5 输出标记摘要

```
📋 大纲标记更新完成

本次设计涉及的元素：
- 【伏笔-埋设】[内容] → 预计第Y章回收
- 【伏笔-回收】[内容] → 来自第X章
- 【套路】[打脸类型/名场面类型]
- 【感情线】[互动类型]

已更新：
✓ 卷X大纲.md - 章节标记
✓ 卷X大纲.md - 伏笔清单

⚠️ 注意：全书记忆将在 auto-write 写完后统一同步
```

## 第六步：级联影响分析（⭐自动触发）

**当场景设计注入大纲后，自动执行以下检查：**

### 6.1 变更程度评估

对比新设计与原大纲，评估变更程度：

| 维度 | 变更程度 | 说明 |
|------|----------|------|
| 核心事件 | 无/轻微/重大 | 主要情节是否改变 |
| 情绪走向 | 无/轻微/重大 | 章节情绪是否改变 |
| 时间线 | 无/轻微/重大 | 时间顺序是否改变 |
| 人物状态 | 无/轻微/重大 | 角色状态是否改变 |
| 章末钩子 | 无/轻微/重大 | 悬念/承接是否改变 |

### 6.2 影响范围预判

```markdown
## 级联影响预判

### 变更类型：[轻微调整 / 中等变更 / 重大重构]

### 受影响章节
- **第X+1章**：[需要调整开头承接 / 无影响]
- **第X+2章**：[可能需要微调 / 无影响]
- **后续章节**：[影响消散于第X+N章 / 全卷可能受影响]

### 伏笔影响
- [伏笔A]：[保留 / 需要移动 / 已删除需重新埋设]
```

### 6.3 自动提示

**如果检测到重大变更（任一维度为"重大"）：**

```
⚠️ 检测到重大变更！

本次设计可能影响后续 N 个章节的承接关系：
- 第X+1章：开头承接断裂
- 第X+2章：时间线不匹配
- ...

建议执行级联更新：
👉 /cascade-update [卷号] [章号]

是否现在执行？[Y/N]
```

**如果检测到轻微变更：**

```
✅ 场景设计已注入。

变更程度：轻微
影响范围：仅本章，后续章节无需调整

可直接使用 /auto-write 进行写作。
```

**如果检测到中等变更：**

```
⚡ 场景设计已注入。

变更程度：中等
建议检查第X+1章的开头承接是否仍然通顺。

可选操作：
- 手动检查后续章节
- 执行 /cascade-update [卷号] [章号] 自动修复
```

---

## 注意事项

1. **创意优先**：本命令鼓励大胆创新，级联更新会帮你处理后续承接
2. **先设计后修复**：专注于设计精彩场景，承接问题交给 cascade-update
3. **保持整体性**：设计时可参考前后章节，但不必受其束缚
4. **迭代优化**：可以多次调用，逐步打磨场景
