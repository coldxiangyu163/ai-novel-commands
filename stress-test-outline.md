# 读者视角压力测试：$ARGUMENTS

**用读者心理模型审视大纲，提前发现可能导致弃书的风险点。**

> **核心理念**：
> - 作者视角 ≠ 读者视角，作者觉得精彩的地方读者可能觉得无聊
> - 读者是"即时满足"的，期待感断裂3章就可能弃书
> - 大纲的目的是让读者"想看下一章"，不是"逻辑完美"

---

## 第一步：读取大纲

1. `卷X-卷名/卷X大纲.md` - 本卷大纲
2. `小说规划.md` - 全书规划（了解目标读者）
3. `爆点曲线.md` - 爆点分布

## 第二步：建立读者心理模型

### 2.1 目标读者画像

```markdown
## 目标读者画像

### 基础信息
- 目标读者群：[男频/女频，年龄段]
- 阅读场景：[通勤/睡前/碎片时间]
- 阅读习惯：[追更/养书/一口气看完]

### 读者心理特征
| 特征 | 描述 | 对大纲的要求 |
|------|------|-------------|
| 即时满足 | 读者想快速获得爽感 | 每章要有小爽点 |
| 期待驱动 | 读者为"想知道后面"而追读 | 每章末要有钩子 |
| 代入需求 | 读者想代入主角 | 主角不能做让读者厌恶的事 |
| 情绪需求 | 读者想获得情绪价值 | 要有明确的情绪满足 |
| 耐心有限 | 读者不会等太久 | 承诺要及时兑现 |

### 读者的核心期待
从本书设定中提取读者最想看的内容：
1. [期待1]：如复仇/变强/打脸
2. [期待2]：如感情线/甜蜜
3. [期待3]：如揭秘/世界观
```

### 2.2 读者阅读心理模型

```markdown
## 读者心理模型

### 追读公式
追读 = 代入感 × 期待感 × 满足感

任何一项为0，读者就会弃书。

### 弃书触发条件
| 触发条件 | 说明 | 容忍阈值 |
|----------|------|----------|
| 期待断裂 | 期待的内容迟迟不出现 | 3-5章 |
| 爽点缺失 | 连续章节无爽点 | 3章 |
| 代入崩塌 | 主角做出让读者讨厌的事 | 1次 |
| 信息过载 | 一次性灌输太多信息 | 1章 |
| 拖沓无聊 | 剧情推进缓慢 | 5章 |
| 逻辑硬伤 | 明显不合理 | 1次严重 |
| 虐而不爽 | 压抑没有释放 | 1卷 |
```

### 风险标准

## 注意事项

1. **读者视角 ≠ 作者视角**：你觉得精彩的地方读者可能觉得无聊
2. **即时满足**：读者的耐心有限，不要考验读者
3. **期待是核心**：读者为"想知道后面"而追读
4. **代入是基础**：一旦代入崩塌，其他都无效
5. **情绪要平衡**：压抑必须有释放

## 附录：读者心理速查

### 读者会因为什么弃书？

| 原因 | 表现 | 容忍度 |
|------|------|--------|
| 无聊 | 没有冲突/爽点 | 3章 |
| 厌恶主角 | 主角圣母/懦弱/脑残 | 1次 |
| 期待落空 | 承诺的剧情不出现 | 5章 |
| 逻辑崩塌 | 明显不合理 | 1次严重 |
| 虐心过度 | 持续压抑无释放 | 1卷 |
| 看不懂 | 信息过载/设定复杂 | 2章 |
| 重复感 | 套路重复/没有新意 | 10章 |

### 读者会因为什么追读？

| 原因 | 表现 | 效果 |
|------|------|------|
| 想看复仇 | 仇人还没死 | 强追读动力 |
| 想看变强 | 还有更高境界 | 持续追读 |
| 想看感情 | 还没在一起 | 甜蜜期待 |
| 想知道真相 | 悬念未揭 | 好奇心驱动 |
| 爽点期待 | 下章要打脸了 | 即时追读 |
