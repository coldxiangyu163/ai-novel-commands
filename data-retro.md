# 数据复盘（赚钱导向）：$ARGUMENTS

目标：把“写得好”变成“卖得好”。

你要做的不是泛泛建议，而是输出**可执行改动**：
- 掉点位置（章节/段落）
- 原因归因（可验证）
- 最小改动清单（按投入产出排序）
- A/B 测试方案（怎么判断改对了）

默认平台：番茄（用户说明起点则按起点偏好调整）。

---

## 0）输入解析（没有完整数据也能复盘）

先解析 `$ARGUMENTS`。

### 0.1 如果用户没贴数据
只问最少信息（最多6问），然后仍然输出“先做这些”的最小改动方案：
1) 当前字数/章节号范围（例如：第1-60章）
2) 最近7天：最明显的变化（收藏/追更/完读/评论热度哪项掉了）
3) 读者吐槽最多的3个词（比如：水、降智、磨叽）
4) 掉点大概发生在第几章附近（如果不知道就说“不清楚”）
5) 你近期做过什么改动（换书名/改简介/剧情转向/断更等）
6) 题材与核心卖点一句话

### 0.2 如果用户贴了数据
把信息结构化成下表（缺项用“未知”标注，不要卡住）：

```markdown
## 复盘输入卡
- 平台：番茄/起点
- 作品阶段：新书前期/中期/后期
- 当前进度：第__章，约__万字
- 最近周期：__天（默认7天）
- 现象：点击↓/进读↓/追更↓/完读↓/评论负面↑/收入↓（可多选）
- 掉点区间：第__-__章（若未知写“未知”）
- 评论高频词TOP5：
- 近期改动：
```

## 1）先做漏斗定位（判断到底哪里坏了）

把问题归到一个主方向（只能选1个主方向，最多2个次方向）：

1) **点击问题**（书名/简介/标签/封面/开篇承诺不清）
2) **留存问题**（读进来了但读不下去：节奏、爽点、冲突、信息密度）
3) **转化问题**（读得下去但不追更/不收藏：章末钩子弱、承诺兑现弱）
4) **合规问题**（被限流/审核风险、敏感词/桥段）

输出：

```markdown
## 漏斗定位结论
- 主问题：
- 次问题：
- 判断依据（用用户给的数据/现象对应）：
```

## 2）掉点章节定位（把“哪里出问题”说清楚）

如果用户提供了章节区间：重点分析该区间。
如果没有：按“新书常见掉点位”推断并给出验证方式。

输出固定格式：

```markdown
## 掉点定位

### 可疑章节列表（按优先级）
| 优先级 | 章节 | 位置 | 可能问题 | 证据/线索 | 建议改动 |
|---:|---|---|---|---|---|
| 1 | 第__章 | 开头200字 |  |  |  |
| 2 | 第__章 | 结尾200字 |  |  |  |

### 快速验证方法
- 验证A：读者是否在“承诺兑现”前离开？（看评论/你自己的阅读感受）
- 验证B：掉点章是否出现“大段解释/日常/重复套路”？
- 验证C：章末是否没有新问题、只有收尾？
```

## 3）归因树（把“为什么掉”说成可操作的原因）

从以下原因里选择最匹配的 1-3 个（必须给出对应证据）：

### 3.1 点击层常见原因
- 类型信号不明确（读者不确定这本爽什么）
- 书名撞/弱（搜不到、记不住、没承诺）
- 简介前两句无钩子

### 3.2 留存层常见原因
- 爽点断供（只铺垫不兑现）
- 冲突失焦（主线停滞、支线喧宾夺主）
- 信息密度失衡（解释太多/信息太少）
- 反派太弱/太蠢（爽感变“降智”）
- 重复供给（连续几章同一种爽点/同一种开头）
- 情绪疲劳（连续高压或连续平淡）

### 3.3 转化层常见原因
- 章末钩子无力（没有“新问题/新危机/新期待”）
- 承诺管理失败（读者等的东西拖太久/兑现太弱）

### 3.4 合规/限流常见原因
- 敏感词/敏感桥段（导致推荐受影响）
- 题材边缘化表达（擦边、暴力、未成年人等高风险）

输出：

```markdown
## 归因结论（可验证）
1) 原因A：…
   - 证据：…
   - 典型表现：…
2) 原因B：…
   - 证据：…

## 我最想优先修的1件事
- 选择：…
- 原因：投入小、回报大/能带动其他问题一起好
```

## 4）最小改动清单（按ROI排序，直接告诉你改哪里）

把改动分成三档：30分钟 / 2小时 / 半天+。

```markdown
## 最小改动清单（ROI排序）

### A档：30分钟内（立刻能做）
1. 改【书名/简介首句/前三章第一段】
   - 改动点：
   - 新版本：
   - 预期影响：点击↑/进读↑

2. 改【掉点章的结尾钩子】（第__章）
   - 把“收尾”改成：危机/反转/新目标
   - 给出可直接替换的200字版本

### B档：2小时内（最划算的正文修复）
1. 重写【第__章某段】（500-1200字）
   - 目的：把爽点兑现做足/把解释压缩成动作与冲突
   - 给出替换文本框架（不必整章重写）

### C档：半天+（只有在B档无效时做）
1. 中期目标重挂牌（第__章起）
   - 新目标：
   - 新规则/新代价：
   - 未来10章节奏表：
```

## 5）A/B测试方案（怎么判断改对了）

番茄常见可测试对象：书名、简介、开篇前500字、章末钩子。

输出：

```markdown
## A/B测试方案

### 测试1：简介首句
- A版：
- B版：
- 成功判据：评论“好看/上头/爽”增加，收藏/追更回升（以你能看到的指标为准）
- 观察窗口：48-72小时

### 测试2：章末钩子
- A版（原）：
- B版（改）：
- 成功判据：该章后续追更/评论热度改善
```
